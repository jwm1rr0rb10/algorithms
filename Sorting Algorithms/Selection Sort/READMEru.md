# Selection Sort (Сортировка выбором)

**Сортировка выбором** работает по принципу "найди минимум и поставь его на правильное место".

---

## Основная идея

- Массив концептуально делится на отсортированную и неотсортированную части.
- На каждом шаге алгоритм находит наименьший элемент в неотсортированной части и меняет его местами с первым элементом этой части (т.е. с элементом, который встанет следующим в отсортированной части).
- Отсортированная часть постепенно растёт слева.

---

## Пример (по возрастанию)

Отсортируем массив: `[10, 7, 8, 9, 1, 5]`

**Шаг 1:**
- Неотсортированная часть: `[10, 7, 8, 9, 1, 5]`
- Минимум: `1` (индекс 4).
- Меняем `10` (индекс 0) и `1` (индекс 4).
- Результат: `[1, 7, 8, 9, 10, 5]`
- Отсортировано: `[1]`, Неотсортировано: `[7, 8, 9, 10, 5]`

**Шаг 2:**
- Неотсортированная часть: `[7, 8, 9, 10, 5]`
- Минимум: `5` (индекс 5).
- Меняем `7` (индекс 1) и `5` (индекс 5).
- Результат: `[1, 5, 8, 9, 10, 7]`
- Отсортировано: `[1, 5]`, Неотсортировано: `[8, 9, 10, 7]`

**Шаг 3:**
- Неотсортированная часть: `[8, 9, 10, 7]`
- Минимум: `7` (индекс 5).
- Меняем `8` (индекс 2) и `7` (индекс 5).
- Результат: `[1, 5, 7, 9, 10, 8]`
- Отсортировано: `[1, 5, 7]`, Неотсортировано: `[9, 10, 8]`

**Шаг 4:**
- Неотсортированная часть: `[9, 10, 8]`
- Минимум: `8` (индекс 5).
- Меняем `9` (индекс 3) и `8` (индекс 5).
- Результат: `[1, 5, 7, 8, 10, 9]`
- Отсортировано: `[1, 5, 7, 8]`, Неотсортировано: `[10, 9]`

**Шаг 5:**
- Неотсортированная часть: `[10, 9]`
- Минимум: `9` (индекс 5).
- Меняем `10` (индекс 4) и `9` (индекс 5).
- Результат: `[1, 5, 7, 8, 9, 10]`
- Отсортировано: `[1, 5, 7, 8, 9]`, Неотсортировано: `[10]`

Последний элемент уже на месте. Массив отсортирован.

## ✅ Реализация на Go

```go
package main

import "fmt"

// selectionSort сортирует массив с помощью алгоритма сортировки выбором
func selectionSort(arr []int) {
	n := len(arr)
	for i := 0; i < n-1; i++ {
		minIndex := i
		for j := i + 1; j < n; j++ {
			if arr[j] < arr[minIndex] {
				minIndex = j
			}
		}
		if minIndex != i {
			arr[i], arr[minIndex] = arr[minIndex], arr[i]
		}
	}
}

func main() {
	data := []int{10, 7, 8, 9, 1, 5}
	fmt.Println("Исходный массив:", data)
	selectionSort(data)
	fmt.Println("Отсортированный массив:", data)

	data2 := []int{4, 2, 7, 1, 9, 3, 6, 8, 5}
	fmt.Println("Исходный массив:", data2)
	selectionSort(data2)
	fmt.Println("Отсортированный массив:", data2)
}
```

## ✅ Реализация на Python

```python
# selection_sort сортирует список с помощью сортировки выбором
def selection_sort(arr):
    n = len(arr)
    for i in range(n - 1):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        if min_idx != i:
            arr[i], arr[min_idx] = arr[min_idx], arr[i]

# Примеры использования сортировки выбором:
data_selection = [10, 7, 8, 9, 1, 5]
print("\nИсходный список (Selection Sort):", data_selection)
selection_sort(data_selection)
print("Отсортированный список (Selection Sort):", data_selection)

data_selection_2 = [4, 2, 7, 1, 9, 3, 6, 8, 5]
print("Исходный список (Selection Sort):", data_selection_2)
selection_sort(data_selection_2)
print("Отсортированный список (Selection Sort):", data_selection_2)
```

## Сложность

### Временная сложность

- **O(n²)** во всех случаях (лучший, средний, худший).
- Внешний цикл выполняется `n - 1` раз.
- Внутренний цикл проходит по оставшейся неотсортированной части (количество сравнений уменьшается).
- Всего примерно `n^2 / 2` сравнений.

### Пространственная сложность

- **O(1)** — сортировка выбором производится на месте, требует только константную дополнительную память.

---

## Почему эти алгоритмы полезны для понимания базовых концепций?

Хотя алгоритмы, такие как Bubble Sort, Insertion Sort и Selection Sort, имеют сложность O(n²), они важны для обучения, потому что:

- **Простота:** Их логика интуитивно понятна и похожа на то, как человек вручную сортирует небольшой набор предметов.
- **Визуализация:** Легко визуализируются, что помогает понять основные идеи сравнения и перемещения элементов.
- **Фундаментальные операции:**  
    - Сравнение и обмен (Bubble Sort)  
    - Сравнение и сдвиг для вставки (Insertion Sort)  
    - Поиск минимума/максимума и обмен (Selection Sort)

Хотя на практике их редко используют для больших данных:

- **Insertion Sort** эффективен для очень маленьких или почти отсортированных массивов.
- **Bubble Sort** и **Selection Sort** почти не применяются, но Selection Sort делает минимальное число обменов — ровно `n - 1` — что может быть важно, если обмен дорог по ресурсам.
