# Обзор алгоритмов сортировки

Алгоритмы сортировки — основа информатики: они используются в индексировании, визуализации данных, работе с БД и многом другом. Каждый алгоритм эффективен для своих типов данных и размерностей. Ниже приведено сравнение и рекомендации по выбору наиболее подходящего алгоритма.

---

## Таблица сравнения алгоритмов

| Алгоритм             | Время (лучшее / ср. / худшее)         | Память      | Стабильность | In-place | Особенности |
|:---------------------|:--------------------------------------:|:-----------:|:------------:|:--------:|:------------|
| **Пузырьковая сортировка** | O(n) / O(n²) / O(n²)               | O(1)        | Да           | Да       | Очень простая, для почти отсортированных данных, учебный пример. |
| **Блочная сортировка**     | O(n + k) (ср.), O(n²) (худш.)      | O(n + k)    | Да           | Нет      | Быстра для равномерных данных, подходит для вещественных чисел. |
| **Сортировка подсчётом**   | O(n + k)                           | O(k)        | Да           | Нет      | Только для целых чисел/ограниченного диапазона, очень быстрая. |
| **Пирамидальная (Heap Sort)** | O(n log n) / O(n log n) / O(n log n) | O(1)    | Нет          | Да       | Всегда O(n log n), неустойчива, хороша при ограниченной памяти. |
| **Вставками**              | O(n) / O(n²) / O(n²)               | O(1)        | Да           | Да       | Отлична для маленьких/почти отсортированных массивов.           |
| **Слиянием (Merge Sort)**  | O(n log n) / O(n log n) / O(n log n) | O(n)      | Да           | Нет      | Стабильна, хороша для списков и внешней сортировки.             |
| **Быстрая (Quick Sort)**   | O(n log n) / O(n log n) / O(n²)    | O(log n)    | Нет          | Да       | На практике самая быстрая для больших массивов, худший случай O(n²) без рандомизации. |
| **Поразрядная (Radix Sort)** | O(nk)                            | O(n + k)    | Да           | Нет      | Некомпаративная, эффективна для целых чисел/строк с ограниченной длиной. |
| **Выбором**                | O(n²) / O(n²) / O(n²)              | O(1)        | Нет          | Да       | Простая, почти всегда хуже сортировки вставками.                 |

## Краткое описание алгоритмов

### **Пузырьковая сортировка**
- **Описание:** Многократное сравнение и обмен соседних элементов.
- **Применение:** Учебные задачи, маленькие/почти отсортированные массивы.
- **Недостатки:** Медленная для больших данных.

### **Блочная сортировка**
- **Описание:** Элементы распределяются по "ведрам", каждое ведро сортируется отдельно.
- **Применение:** Равномерно распределённые или вещественные данные.
- **Недостатки:** Не подходит для неравномерных данных.

### **Сортировка подсчётом**
- **Описание:** Подсчёт количества вхождений каждого элемента.
- **Применение:** Целые числа или объекты с небольшим известным диапазоном ключей.
- **Недостатки:** Большой диапазон ключей = много памяти.

### **Пирамидальная сортировка (Heap Sort)**
- **Описание:** Строит кучу и последовательно извлекает максимум/минимум.
- **Применение:** In-place сортировка при необходимости O(n log n), если стабильность не нужна.
- **Недостатки:** Неустойчива, медленнее быстрой сортировки на практике.

### **Сортировка вставками**
- **Описание:** Строит отсортированный массив, вставляя элементы на место.
- **Применение:** Малые или почти отсортированные массивы, онлайн-сортировка.
- **Недостатки:** Медленная на больших/случайных данных.

### **Сортировка слиянием**
- **Описание:** Рекурсивно делит и сливает списки по порядку.
- **Применение:** Связные списки, внешняя сортировка, нужна стабильность.
- **Недостатки:** Требует O(n) памяти.

### **Быстрая сортировка**
- **Описание:** Разделяет массив на части по опорному элементу.
- **Применение:** Универсальный выбор для больших массивов.
- **Недостатки:** Неустойчива, худший случай O(n²) (устраняется рандомизацией).

### **Поразрядная сортировка**
- **Описание:** Сортирует поразрядно (поразрядно по цифрам/буквам).
- **Применение:** Большие списки целых чисел, строк, коротких ключей.
- **Недостатки:** Не применяется для вещественных чисел без модификаций.

### **Сортировка выбором**
- **Описание:** Находит минимум (или максимум) и ставит на место.
- **Применение:** Малые массивы, учебные задачи.
- **Недостатки:** Неэффективна для больших данных, неустойчива.

---

## Практические советы

- **Малые/почти отсортированные массивы:** Сортировка вставками.
- **Большие случайные массивы:** Быстрая или сортировка слиянием (если нужна стабильность).
- **Требуется стабильность:** Merge, Bubble, Counting, Radix Sort.
- **Ограниченная память:** Heap, Insertion или Selection Sort.
- **Ключи — целые числа/ограниченный диапазон:** Counting или Radix Sort.
- **Связные списки:** Merge Sort.
