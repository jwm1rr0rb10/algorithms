# Merge Sort (Сортировка слиянием)

**Сортировка слиянием (Merge Sort)** — это классический алгоритм сортировки на основе принципа "разделяй и властвуй", аналогичный быстрой сортировке (Quick Sort), но с другой логикой. Алгоритм разработан Джоном фон Нейманом.

---

## Основная идея

- **Разделение:** Рекурсивно делить массив на две половины, пока каждая подгруппа не станет содержать по одному элементу (одиночный элемент считается отсортированным).
- **Преодоление (Conquer):** Сортировка подмассивов происходит неявно — за счёт рекурсии до разбиения на одиночные элементы.
- **Слияние:** Постепенно объединять отсортированные подмассивы обратно в большие отсортированные массивы до полного слияния исходного массива.

Ключевая операция — слияние двух уже отсортированных подмассивов в один.

---

## Шаг "Слияния"

Допустим, у нас есть два отсортированных подмассива:
- Левый: `[1, 5, 8]`
- Правый: `[2, 7, 10]`

Чтобы объединить их в один отсортированный массив:

1. Создать пустой результирующий массив (или использовать вспомогательный).
2. Использовать два указателя (индекса) — по одному на каждый подмассив, начиная с начала.
3. Сравнивать элементы, на которые указывают указатели.
4. Копировать меньший элемент в результат и продвигать соответствующий указатель.
5. Повторять шаги 3–4, пока один из подмассивов не закончится.
6. Скопировать оставшиеся элементы из другого подмассива (они уже отсортированы).

**Пример слияния:**

Левый: `[1, 5, 8]`, указатель `i` на `1`  
Правый: `[2, 7, 10]`, указатель `j` на `2`  
Результат: `[]`  
- Сравниваем `1` и `2`. Копируем `1` → `[1]`
- Сравниваем `5` и `2`. Копируем `2` → `[1, 2]`
- Сравниваем `5` и `7`. Копируем `5` → `[1, 2, 5]`
- Сравниваем `8` и `7`. Копируем `7` → `[1, 2, 5, 7]`
- Сравниваем `8` и `10`. Копируем `8` → `[1, 2, 5, 7, 8]`
- Левый закончился, копируем остаток `10` → `[1, 2, 5, 7, 8, 10]`

---

## Процесс "Разделения"

Пример массива: `[10, 7, 8, 9, 1, 5]`

1. Делим: `[10, 7, 8]` и `[9, 1, 5]`
2. Делим `[10, 7, 8]`: `[10]` и `[7, 8]`
3. Делим `[7, 8]`: `[7]` и `[8]`
4. Делим `[9, 1, 5]`: `[9]` и `[1, 5]`
5. Делим `[1, 5]`: `[1]` и `[5]`

Далее объединяем на обратном пути:
- Сливаем `[7]` и `[8]` → `[7, 8]`
- Сливаем `[1]` и `[5]` → `[1, 5]`
- Сливаем `[10]` и `[7, 8]` → `[7, 8, 10]`
- Сливаем `[9]` и `[1, 5]` → `[1, 5, 9]`
- Сливаем `[7, 8, 10]` и `[1, 5, 9]` → `[1, 5, 7, 8, 9, 10]`

## Реализация на Go

Сортировка слиянием обычно требует вспомогательного массива для слияния.

```go
package main

import "fmt"

// mergesort — основная функция, вызывающая рекурсивную сортировку
func mergesort(arr []int) []int {
	if len(arr) <= 1 {
		return arr // Базовый случай: уже отсортировано
	}

	mid := len(arr) / 2
	left := mergesort(arr[:mid])
	right := mergesort(arr[mid:])

	return merge(left, right)
}

// merge — объединяет два отсортированных среза
func merge(left, right []int) []int {
	result := make([]int, 0, len(left)+len(right))
	i, j := 0, 0

	for i < len(left) && j < len(right) {
		if left[i] <= right[j] {
			result = append(result, left[i])
			i++
		} else {
			result = append(result, right[j])
			j++
		}
	}

	for i < len(left) {
		result = append(result, left[i])
		i++
	}
	for j < len(right) {
		result = append(result, right[j])
		j++
	}

	return result
}

func main() {
	data := []int{10, 7, 8, 9, 1, 5}
	fmt.Println("Исходный массив:", data)
	sortedData := mergesort(data)
	fmt.Println("Отсортированный массив:", sortedData)

	data2 := []int{4, 2, 7, 1, 9, 3, 6, 8, 5}
	fmt.Println("Исходный массив:", data2)
	sortedData2 := mergesort(data2)
	fmt.Println("Отсортированный массив:", sortedData2)
}
```

---

## Разбор кода (Go)

- `mergesort(arr []int) []int`: Главная рекурсивная функция. Если массив пуст или состоит из одного элемента, возвращает его как есть. Иначе делит массив, рекурсивно сортирует каждую часть, затем сливает их.
- `merge(left, right []int) []int`: Сливает два отсортированных среза в один отсортированный.

---

## Сложность Merge Sort

- **Время:** O(n log n) в лучшем, худшем и среднем случаях (log n уровней деления, каждый шаг слияния — O(n)).
- **Память:** O(n) из-за вспомогательного массива для слияния.

---

## Преимущества сортировки слиянием

- Гарантированная сложность O(n log n) во всех случаях.
- Стабильная сортировка (сохраняет относительный порядок равных элементов).

---

## Недостатки сортировки слиянием

- Требует O(n) дополнительной памяти для вспомогательного массива.
- Может быть медленнее быстрой сортировки на практике (из-за копирования), хотя худший случай лучше.

## Реализация на Python

```python
# Функция для слияния двух отсортированных списков
def merge(left, right):
    result = []
    i, j = 0, 0

    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1

    while i < len(left):
        result.append(left[i])
        i += 1

    while j < len(right):
        result.append(right[j])
        j += 1

    return result

# Главная рекурсивная функция сортировки слиянием
def mergesort(arr):
    if len(arr) <= 1:
        return arr

    mid = len(arr) // 2
    left = mergesort(arr[:mid])
    right = mergesort(arr[mid:])

    return merge(left, right)

# Примеры использования:
data = [10, 7, 8, 9, 1, 5]
print("Исходный список:", data)
sorted_data = mergesort(data)
print("Отсортированный список:", sorted_data)

data2 = [4, 2, 7, 1, 9, 3, 6, 8, 5]
print("Исходный список:", data2)
sorted_data2 = mergesort(data2)
print("Отсортированный список:", sorted_data2)
```
