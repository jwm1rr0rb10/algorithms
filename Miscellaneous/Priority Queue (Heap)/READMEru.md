# 🔍 Приоритетная Очередь (Куча)

## 📌 Описание Проблемы

Представьте себе не обычную очередь, где первый пришедший первым обслуживается (FIFO - First-In, First-Out), а очередь, где **важен приоритет**. Например, в отделении скорой помощи пациент с сердечным приступом обслуживается раньше пациента со сломанным пальцем, независимо от того, кто пришел первым.

**Приоритетная очередь (Priority Queue)** — это абстрактный тип данных, который поддерживает коллекцию элементов, каждый из которых имеет ассоциированный *приоритет*. В отличие от обычной очереди, элементы извлекаются не в порядке их добавления, а **в порядке их приоритета**.

Основная операция приоритетной очереди — это извлечение элемента с *наивысшим* (или *наименьшим*) приоритетом.

---

## 💡 Идея и Подход

Наиболее распространенной и эффективной реализацией приоритетной очереди является использование **кучи (Heap)**. Куча — это специализированная древовидная структура данных, которая удовлетворяет так называемому "свойству кучи".

### Что такое Куча (Heap)?

Куча — это **полное бинарное дерево (Complete Binary Tree)**, которое удовлетворяет свойству кучи.

* **Полное бинарное дерево:** Это бинарное дерево, в котором все уровни, кроме, возможно, последнего, полностью заполнены, а узлы на последнем уровне расположены как можно левее. Это свойство очень важно, так как позволяет эффективно хранить кучу в **массиве**, без использования явных указателей, экономя память и улучшая локальность данных.

* **Свойство кучи:** Определяет порядок элементов в дереве:

    * **Max-Heap (Макс-куча):** Для каждого узла `N`, значение `N` **больше или равно** значениям его дочерних узлов. Следовательно, самый большой элемент всегда находится в корне кучи.

    * **Min-Heap (Мин-куча):** Для каждого узла `N`, значение `N` **меньше или равно** значениям его дочерних узлов. Следовательно, самый маленький элемент всегда находится в корне кучи.

### Как работает Куча (и Приоритетная Очередь)?

Куча использует две основные операции для поддержания своего свойства после вставки или удаления:

1.  **"Всплывание" (Heapify-Up / Sift-Up / Bubble-Up):**

    * Используется после **вставки** нового элемента. Новый элемент добавляется в первое свободное место в конце массива (последний лист в дереве).

    * Если этот новый элемент нарушает свойство кучи (например, в Min-Heap новый элемент меньше своего родителя), он **меняется местами со своим родителем** до тех пор, пока свойство кучи не восстановится, или пока он не достигнет корня.

2.  **"Просеивание вниз" (Heapify-Down / Sift-Down / Bubble-Down):**

    * Используется после **удаления** элемента (обычно корневого). Корень удаляется, и его место занимает последний элемент кучи (последний элемент массива).

    * Если этот элемент нарушает свойство кучи (например, в Min-Heap, он больше своих дочерних элементов), он **меняется местами со своим наименьшим (в Min-Heap) или наибольшим (в Max-Heap) дочерним элементом** до тех пор, пока свойство кучи не восстановится, или пока он не станет листом.

Эти операции "всплывания" и "просеивания вниз" гарантируют, что куча всегда остается сбалансированной (высота $O(\log n)$), и свойство кучи сохраняется.

### Представление в Массиве:

Благодаря свойству полного бинарного дерева, кучу можно очень эффективно хранить в массиве. Для узла по индексу `i` (в массиве с 0-й индексацией):

* **Левый дочерний элемент:** `2 * i + 1`

* **Правый дочерний элемент:** `2 * i + 2`

* **Родительский элемент:** `(i - 1) // 2` (используя целочисленное деление)

Это избавляет от необходимости хранить указатели, делая реализацию простой и память более эффективной.

## 🧪 Пример на Python (Min-Heap)

В Python модуль `heapq` реализует алгоритм min-heap. `priority_queue` в C++ STL по умолчанию является max-heap. `java.util.PriorityQueue` в Java по умолчанию является min-heap.

```python
import heapq

print("--- Пример Приоритетной Очереди (Min-Heap) ---")

# Инициализация пустой min-heap (список в Python, к которому применяются функции heapq)
min_heap = []

# 1. Вставка элементов (heappush):
# heapq.heappush(heap, item) вставляет элемент, поддерживая свойство min-heap
print("Вставка элементов:")
heapq.heappush(min_heap, 10) # [10]
print(f"После 10: {min_heap}")

heapq.heappush(min_heap, 30) # [10, 30]
print(f"После 30: {min_heap}")

heapq.heappush(min_heap, 20) # [10, 30, 20] -> Python's internal representation might be slightly different, but the heap property holds
print(f"После 20: {min_heap}")

heapq.heappush(min_heap, 5) # [5, 10, 20, 30] (internally balanced)
print(f"После 5: {min_heap}")

heapq.heappush(min_heap, 25)
print(f"После 25: {min_heap}")

print(f"\nТекущее состояние Min-Heap (внутреннее представление): {min_heap}")
# Обратите внимание: массив не будет полностью отсортирован, но root (индекс 0) всегда будет наименьшим.

# 2. Извлечение наименьшего элемента (heappop):
# heapq.heappop(heap) извлекает и возвращает наименьший элемент, затем перебалансирует кучу.
print("\nИзвлечение элементов:")
smallest = heapq.heappop(min_heap) # Извлекает 5
print(f"Извлечен наименьший элемент: {smallest}")
print(f"Heap после извлечения: {min_heap}") # [10, 25, 20, 30] (внутреннее представление)

smallest = heapq.heappop(min_heap) # Извлекает 10
print(f"Извлечен наименьший элемент: {smallest}")
print(f"Heap после извлечения: {min_heap}") # [20, 25, 30]

# 3. Просмотр наименьшего элемента (peek/top):
# В Python heapq нет прямой функции peek/top, но это всегда элемент с индексом 0
if min_heap:
    print(f"\nНаименьший элемент (без удаления): {min_heap[0]}")
else:
    print("\nКуча пуста.")

# 4. Построение кучи из существующего списка (heapify):
print("\nПостроение кучи из существующего списка:")
my_list = [9, 1, 7, 3, 5]
heapq.heapify(my_list) # Преобразует список в min-heap "на месте"
print(f"Список после heapify: {my_list}") # [1, 3, 7, 9, 5] (внутреннее представление min-heap)

# Извлечение из созданной кучи
print(f"Извлечен первый: {heapq.heappop(my_list)}") # Извлекает 1
print(f"Извлечен второй: {heapq.heappop(my_list)}") # Извлекает 3

# Пример Max-Heap (не напрямую, но можно имитировать, храня отрицательные значения)
print("\n--- Пример имитации Max-Heap (с использованием Min-Heap) ---")
max_heap_simulated = []

heapq.heappush(max_heap_simulated, -10)
heapq.heappush(max_heap_simulated, -30)
heapq.heappush(max_heap_simulated, -20)
heapq.heappush(max_heap_simulated, -5)

print(f"Simulated Max-Heap (отрицательные значения): {max_heap_simulated}")

# Извлекаем "наименьшее" отрицательное, что соответствует "наибольшему" положительному
largest_simulated = -heapq.heappop(max_heap_simulated)
print(f"Извлечен наибольший элемент (симулированный): {largest_simulated}") # 30
print(f"Simulated Max-Heap после извлечения: {max_heap_simulated}")
```

## ⏱️ Сложность

Основное преимущество кучи - логарифмическая временная сложность для ключевых операций.

* **Временная Сложность (Средний и Худший Случай):**
    * **Вставка (`push`/`add`):** $O(\log n)$ - из-за операции "всплывания", которая проходит по высоте дерева.
    * **Удаление (`pop`/`extract_min/max`):** $O(\log n)$ - из-за операции "просеивания вниз", которая также проходит по высоте дерева.
    * **Просмотр наивысшего/наименьшего элемента (`peek`/`top`):** $O(1)$ - элемент всегда находится в корне.
    * **Построение кучи из массива (`heapify`):** $O(n)$ - хотя кажется, что каждый элемент требует $O(\log n)$, агрегированная стоимость всех "просеиваний вниз" при построении кучи составляет $O(n)$.
    * **Изменение приоритета (`decrease_key`/`increase_key`):** $O(\log n)$ - нахождение элемента $O(n)$ (без дополнительной хэш-таблицы), затем "всплывание" или "просеивание вниз". Если у нас есть прямой доступ к элементу (например, через индекс или указатель), то $O(\log n)$.

* **Пространственная Сложность:**
    * $O(n)$
    Структура данных должна хранить все $n$ элементов. Если она реализована в массиве, то это очень эффективно с точки зрения использования памяти, так как нет накладных расходов на указатели, как в обычных древовидных структурах.

## ⚠️ Соображения

* **Доступ по индексу:** В отличие от отсортированных массивов или списков, прямой доступ к элементу по его значению или произвольному индексу (кроме корня) не является эффективным ($O(N)$ в худшем случае), так как куча является "частично отсортированной", а не полностью отсортированной.
* **Гибкость:** Приоритетные очереди идеально подходят, когда вам нужен *только* доступ к элементу с наивысшим/наименьшим приоритетом, а не к полному отсортированному списку всех элементов.
* **Имитация Max-Heap:** В языках, где по умолчанию есть только Min-Heap (как в Python `heapq`), Max-Heap можно легко имитировать, вставляя элементы с инвертированным знаком (например, `-value`) и затем инвертируя знак при извлечении.
* **Выбор между Min-Heap и Max-Heap:** Зависит от того, нужно ли вам извлекать наименьший или наибольший элемент как приоритетный.

## 🧭 Применения

Приоритетные очереди (и кучи) являются фундаментальными для множества алгоритмов и систем:

* **Алгоритмы на графах:**
    * **Алгоритм Дейкстры** (Dijkstra's Algorithm): Поиск кратчайшего пути от одной вершины до всех других в графе с неотрицательными весами рёбер. Приоритетная очередь используется для эффективного выбора следующей вершины с наименьшим текущим расстоянием.
    * **Алгоритм Прима** (Prim's Algorithm) и **Алгоритм Крускала** (Kruskal's Algorithm): Поиск минимального остовного дерева в графе. Приоритетная очередь помогает выбирать рёбра с наименьшим весом.
* **Операционные системы:**
    * **Планирование задач/процессов:** Управление очередью процессов, ожидающих выполнения, где более приоритетные задачи (например, системные) выполняются раньше менее приоритетных (например, фоновые).
    * **Управление прерываниями:** Обработка системных прерываний на основе их приоритета.
* **Моделирование событий:** В дискретно-событийных симуляциях события хранятся в приоритетной очереди, упорядоченные по времени их наступления, что позволяет обрабатывать их в правильной хронологической последовательности.
* **Сжатие данных:**
    * **Кодирование Хаффмана** (Huffman Coding): Использует min-heap для построения оптимального префиксного кода, выбирая символы с наименьшей частотой.
* **Сортировка:**
    * **Пирамидальная сортировка** (Heapsort): Эффективный алгоритм сортировки на основе кучи со сложностью $O(N \log N)$.
* **Системы искусственного интеллекта:**
    * **Алгоритм A*** (A\* Search Algorithm): Поиск пути, который использует приоритетную очередь для хранения узлов, упорядоченных по оценочной общей стоимости пути.
* **Сетевое оборудование:** Управление очередями пакетов на маршрутизаторах, где высокоприоритетный трафик (например, голосовой) должен быть передан быстрее.
* **Нахождение K-го наибольшего/наименьшего элемента:** Можно использовать min-heap или max-heap для эффективного нахождения K-го наибольшего или наименьшего элемента в потоке данных или большом наборе данных.

## 🧩 Связь с LeetCode

Приоритетные очереди и кучи являются одними из самых часто используемых структур данных на LeetCode и в алгоритмических соревнованиях. Они незаменимы в задачах, где:

* Вам нужно **постоянно извлекать элемент с наивысшим/наименьшим приоритетом**.
* Вы имеете дело с **потоком данных**, и вам нужно поддерживать отсортированный поднабор (например, `k` наибольших/наименьших элементов).
* Вам необходимо реализовать алгоритмы, основанные на **жадных стратегиях** или **кратчайших путях/минимальных остовных деревьях**.
* Такие задачи, как **медиана скользящего окна, объединение K отсортированных списков, планирование задач** или **топологическая сортировка** часто требуют приоритетных очередей.
* Ограничения по времени ($O(N \log N)$ или $O(\log N)$ на операцию) часто указывают на кучу как на подходящее решение.

Понимание и умение эффективно использовать кучи - это ключевой навык для решения множества алгоритмических задач.
