# 🔍 Дек (двусторонняя очередь)

## 📌 Описание задачи

Дек (двусторонняя очередь) — это линейная структура данных, обобщающая обычную очередь. Элементы могут добавляться или удаляться как с начала (головы), так и с конца (хвоста). В отличие от стандартной очереди (FIFO), где добавление происходит с конца, а удаление — с начала, и стека (LIFO), где элементы добавляются и удаляются с одного конца, дек предоставляет гибкость операций с обеих сторон.

## 💡 Идея и подход

Основная идея дека — обеспечить **эффективность O(1)** для операций добавления и удаления элементов с **обоих концов**. Это достигается следующими реализациями:

* **Двусвязный список**: Каждый узел хранит ссылки на предыдущий и следующий элементы, что позволяет быстро вставлять и удалять с обеих сторон.
* **Кольцевой массив (динамический массив с обёрнутыми индексами)**: Используется фиксированный массив, но индексы могут «оборачиваться» при достижении границ. При переполнении происходит увеличение размера.

### Основные операции:

* `append()` / `push_back()`: Добавить элемент в конец.
* `appendleft()` / `push_front()`: Добавить элемент в начало.
* `pop()` / `pop_back()`: Удалить и вернуть элемент с конца.
* `popleft()` / `pop_front()`: Удалить и вернуть элемент с начала.
* `peek()` / `front()` / `back()`: Получить (не удаляя) первый или последний элемент.
* `is_empty()`: Проверить, пуст ли дек.
* `size()`: Количество элементов.

## 🧪 Пример на Python

```python
from collections import deque

print("--- Пример Deque (двусторонняя очередь) ---")

# 1. Инициализация
my_deque = deque()

# 2. Добавление элементов
my_deque.append(10)      # В конец: deque([10])
my_deque.append(20)      # В конец: deque([10, 20])
my_deque.appendleft(5)   # В начало: deque([5, 10, 20])
my_deque.appendleft(1)   # В начало: deque([1, 5, 10, 20])

print(f"Deque после добавлений: {my_deque}")
print(f"Размер: {len(my_deque)}")

# 3. Доступ без удаления
print(f"Первый элемент: {my_deque[0]}")
print(f"Последний элемент: {my_deque[-1]}")

# 4. Удаление элементов
removed_right = my_deque.pop()  # Удалить с конца
print(f"Удалено справа: {removed_right}, Deque: {my_deque}")

removed_left = my_deque.popleft()  # Удалить с начала
print(f"Удалено слева: {removed_left}, Deque: {my_deque}")

# 5. Проверка на пустоту
print(f"Пуст ли deque? {not my_deque}")

# 6. Добавление нескольких элементов
my_deque.extend([30, 40])       # В конец
my_deque.extendleft([-1, 0])    # В начало
print(f"После extend: {my_deque}")

# 7. Поворот (вращение)
my_deque.rotate(1)  # Сдвиг вправо на 1
print(f"После поворота вправо: {my_deque}")

my_deque.rotate(-2) # Сдвиг влево на 2
print(f"После поворота влево: {my_deque}")

# 8. Очистка
my_deque.clear()
print(f"После очистки: {my_deque}")
```

## ⏱️ Сложность

Для эффективной реализации (как в `collections.deque`):

* `append()`, `appendleft()`: `O(1)`
* `pop()`, `popleft()`: `O(1)`
* Доступ по индексу (`deque[i]`): `O(n)` (в Python — `O(k)`, где `k` — расстояние до ближайшего конца)
* `len()`: `O(1)`
* `rotate(k)`: `O(k)`

**Пространственная сложность**: `O(n)`

## ⚠️ Особенности

* **Случайный доступ**: Менее эффективен, чем в списке (`list`). Лучше использовать при частых операциях с концами.
* **Память**: В отличие от массивов, дек (при реализации списком) не использует непрерывную память.
* **Выбор структуры**:

  * Частые вставки/удаления с двух концов → Deque
  * Только с конца → Stack
  * Только с начала и конца → Queue
  * Частый доступ по индексу → list/array

## 🧭 Применения

* **Скользящее окно**: Определение максимумов/минимумов в подмассиве.
* **Имитация стека или очереди**: Быстрые операции с обоих концов.
* **Поиск в ширину (BFS)**: Можно использовать `appendleft` в вариантах (например, 0-1 BFS).
* **История действий (undo/redo)**
* **Монотонные очереди/стеки**: Упорядоченное хранение с добавлением/удалением с концов.
* **История браузера**
* **Work-stealing**: В параллельных алгоритмах
