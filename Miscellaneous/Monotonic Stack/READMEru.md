# 🔍 Монотонный Стек (Monotonic Stack)

## 📌 Описание Проблемы

**Монотонный стек** — это структура данных типа "стек", в которой элементы поддерживаются в строго возрастающем или строго убывающем порядке. Это не отдельная структура данных, как обычная очередь или дек, а скорее **техника** или **шаблон** использования стандартного стека (LIFO) для эффективного решения задач, связанных с поиском "следующего большего элемента", "следующего меньшего элемента" или схожих концепций внутри массива или последовательности.

Основная задача, которую он решает, — это эффективная обработка элементов один за другим и использование стека для поддержания подмножества ранее обработанных элементов, которые являются "кандидатами" для выполнения определенных монотонных условий для будущих элементов. Поддерживая стек монотонным, мы можем выполнять эти поиски и обновления за амортизированное время $O(1)$ на каждый элемент.

---

## 💡 Идея и Подход

Общий подход к использованию Монотонного Стека включает итерацию по входному массиву (обычно слева направо, но иногда справа налево, в зависимости от задачи) и выполнение определенных действий на основе текущего элемента и верхнего элемента стека.

### Для Монотонно Возрастающего Стека:

(Используется для поиска **следующего меньшего элемента** или **предыдущего меньшего элемента** для элементов)
Стек хранит элементы (или их индексы) в возрастающем порядке от основания до вершины.

1.  **Итерация по массиву (например, `nums[i]`):**
2.  **Пока стек не пуст И элемент на вершине стека (`nums[stack.top()]`) больше или равен `nums[i]`:**
    * Это означает, что `nums[i]` является меньшим элементом, который "нашел" свой следующий меньший элемент (если обработка идет слева направо) или будет предыдущим меньшим элементом для `nums[i]`.
    * Извлекайте (`pop`) элементы из стека. Для каждого извлеченного элемента `nums[i]` является его "следующим меньшим элементом".
3.  **Поместите `i` в стек:** `nums[i]` теперь является кандидатом на роль "следующего меньшего элемента" для будущих элементов.

### Для Монотонно Убывающего Стека:

(Используется для поиска **следующего большего элемента** или **предыдущего большего элемента** для элементов)
Стек хранит элементы (или их индексы) в убывающем порядке от основания до вершины.

1.  **Итерация по массиву (например, `nums[i]`):**
2.  **Пока стек не пуст И элемент на вершине стека (`nums[stack.top()]`) меньше или равен `nums[i]`:**
    * Это означает, что `nums[i]` является большим элементом, который "нашел" свой следующий больший элемент (если обработка идет слева направо) или будет предыдущим большим элементом для `nums[i]`.
    * Извлекайте (`pop`) элементы из стека. Для каждого извлеченного элемента `nums[i]` является его "следующим большим элементом".
3.  **Поместите `i` в стек:** `nums[i]` теперь является кандидатом на роль "следующего большего элемента" для будущих элементов.

### Основные Преимущества:

* **Линейное Время:** Каждый элемент помещается в стек не более одного раза и извлекается из стека не более одного раза, что приводит к общей временной сложности $O(N)$.
* **Эффективный Поиск:** Обеспечивает доступ за константное время к наиболее актуальному элементу (вершине стека), который поддерживает монотонность.

## 🧪 Пример на Python

Классическая задача, решаемая с помощью монотонного стека, — это "Следующий больший элемент I" (LeetCode 496) или "Следующий больший элемент II" (LeetCode 503 для циклических массивов), а также "Наибольший прямоугольник в гистограмме" (LeetCode 84).

Вот пример для поиска "Следующего большего элемента" для каждого элемента в массиве с использованием Монотонно Убывающего Стека:

```python
from collections import deque

def next_greater_element(nums):
    """
    Находит следующий больший элемент для каждого элемента во входном списке.
    Если больший элемент справа не существует, используется -1.

    Использует Монотонно Убывающий Стек.
    Стек хранит индексы элементов в порядке убывания их значений.
    Когда обрабатывается новый элемент `nums[i]`, любой элемент на вершине стека,
    который меньше `nums[i]`, нашел свой следующий больший элемент.
    """
    n = len(nums)
    # Инициализируем результирующий массив значением -1 (по умолчанию для отсутствия следующего большего элемента)
    result = [-1] * n
    # Стек хранит индексы
    stack = [] # Монотонно Убывающий Стек (хранит индексы элементов в порядке убывания значения)

    for i in range(n):
        # Пока стек не пуст И элемент на вершине стека
        # МЕНЬШЕ текущего элемента (nums[i]):
        # Это означает, что nums[i] является Следующим Большим Элементом для stack.top()
        while stack and nums[stack[-1]] < nums[i]:
            idx_to_update = stack.pop()
            result[idx_to_update] = nums[i]

        # Помещаем индекс текущего элемента в стек.
        # Он является кандидатом на роль Следующего Большего Элемента для будущих элементов.
        stack.append(i)

    # После итерации, любые элементы, оставшиеся в стеке, не имеют следующего большего элемента
    # в массиве (или мы завершили обработку текущего прохода для задач с циклическими массивами).
    # Они останутся -1 в результате, как было инициализировано.

    return result

# Пример использования:
nums1 = [4, 5, 2, 10, 8]
print(f"Nums: {nums1} -> Следующие Большие Элементы: {next_greater_element(nums1)}")
# Ожидаемый результат: [5, 10, 10, -1, -1]
# Пояснение:
# 4 -> 5
# 5 -> 10
# 2 -> 10
# 10 -> -1
# 8 -> -1

nums2 = [1, 3, 2, 4]
print(f"Nums: {nums2} -> Следующие Большие Элементы: {next_greater_element(nums2)}")
# Ожидаемый результат: [3, 4, 4, -1]

nums3 = [13, 7, 6, 12]
print(f"Nums: {nums3} -> Следующие Большие Элементы: {next_greater_element(nums3)}")
# Ожидаемый результат: [-1, 12, 12, -1]
```

## ⏱️ Сложность

Метод монотонного стека обеспечивает эффективную производительность.

* **Временная Сложность:** $O(N)$, где $N$ — количество элементов во входном массиве.
    * Каждый элемент помещается в стек не более одного раза.
    * Каждый элемент извлекается из стека не более одного раза.
    * Следовательно, общее количество операций push и pop пропорционально $N$.
* **Пространственная Сложность:** $O(N)$ в худшем случае.
    * Если входной массив строго монотонный (например, `[1, 2, 3, 4, 5]` для убывающего стека или `[5, 4, 3, 2, 1]` для возрастающего стека), все элементы могут быть помещены в стек до того, как какой-либо из них будет извлечен, что приведет к тому, что стек будет содержать до $N$ элементов.

## ⚠️ Соображения

* **Проверка Пустого Стека:** Всегда проверяйте, пуст ли стек, прежде чем пытаться получить доступ к `stack[-1]` (верхний элемент).
* **Направление Итерации:** Направление итерации (слева направо против справа налево) и тип монотонного стека (возрастающий против убывающего) зависят от того, ищете ли вы "следующий" или "предыдущий" элемент и "больший" или "меньший". Часто бывает полезно нарисовать небольшие примеры, чтобы определить правильную настройку.
    * Для поиска **Следующего Большего Элемента**: Итерация Слева->Направо, Монотонно Убывающий Стек.
    * Для поиска **Предыдущего Большего Элемента**: Итерация Слева->Направо, Монотонно Возрастающий Стек (pop, если текущий `>=` stack.top(), результат для текущего - stack.top()).
    * И так далее.
* **Обработка Дубликатов:** В зависимости от задачи может потребоваться скорректировать сравнение (например, `>` против `>=`), чтобы включать или исключать элементы, равные текущему.
* **Круговые Массивы:** Для задач, включающих круговые массивы, часто требуется итерировать по массиву дважды (например, `nums + nums` или `range(2 * n)`), чтобы имитировать цикличность.

## 🧭 Применения

Монотонные Стеки — это специализированная, но очень мощная техника для решения ряда задач, где требуется найти элементы, связанные с "следующим/предыдущим большим/меньшим":

* **Следующий Больший/Меньший Элемент:** Поиск первого элемента справа (или слева), который больше (или меньше) текущего элемента.
* **Наибольший Прямоугольник в Гистограмме:** Поиск максимальной площади прямоугольника, который может быть образован внутри гистограммы. Это классическое применение, где монотонный стек используется для эффективного нахождения левой и правой границ для каждого столбца.
* **Сумма Минимумов/Максимумов Подмассивов:** Вычисление суммы минимумов (или максимумов) всех возможных смежных подмассивов.
* **Сбор Дождевой Воды (Trapping Rain Water):** Хотя задача может быть решена с помощью двух указателей, монотонный стек также может быть использован.
* **Удаление K Цифр для Получения Наименьшего Числа:** Используется для поиска лексикографически наименьшего числа после удаления k цифр.
* **Правильные Скобки / Самая Длинная Правильная Последовательность Скобок:** Хотя это в первую очередь задачи со стеком, некоторые вариации могут использовать монотонность.
* **Задача о Диапазоне Акций (Stock Span Problem):** Вычисление диапазона для цены акций каждого дня (количество последовательных дней до текущего дня, когда цена была меньше или равна цене текущего дня).

Задачи с Монотонным Стеком очень распространены на собеседованиях и в спортивном программировании, часто попадая в категории сложности "средний" и "сложный", потому что они требуют концептуального перехода от базового использования стека.

* **Прямое Применение:** Задачи типа "Следующий больший элемент" (I и II), "Следующий меньший элемент", "Предыдущий больший/меньший элемент".

* **Распознавание Паттернов:** Ключ к успеху — это распознать, когда задача просит вас найти "первый элемент слева/справа, который удовлетворяет определенному условию (больше/меньше)". Если условие включает относительные значения и порядок, монотонный стек часто является оптимальным решением.

Овладение техникой монотонного стека — это значительный шаг в расширении вашего алгоритмического инструментария за пределы базовых структур данных.
