# 🔍 Монотонная Очередь (Monotonic Queue)

## 📌 Описание задачи

**Монотонная очередь** (Monotonic Queue), также известная как "дек с монотонностью" или "максимум/минимум в скользящем окне", — это специализированная структура данных, которая поддерживает элементы в **строго возрастающем** или **строго убывающем** порядке.

Это не самостоятельная структура, а **паттерн** использования двухсторонней очереди (`deque`) для эффективного решения задач со **скользящим окном**.

Данный подход обеспечивает **O(1)** доступ к минимуму или максимуму в окне, а общая сложность становится **O(N)** вместо **O(N·K)** или **O(N·logK)**.

---

## 💡 Идея и подход

### Основные принципы:

- Используется `deque` для хранения **индексов**, а не значений.
- Поддерживается **монотонный порядок**:
  - Убывающий — для поиска **максимума**
  - Возрастающий — для поиска **минимума**
- "Бесполезные" элементы удаляются заблаговременно.

### Монотонно убывающая очередь (для поиска максимума):

```text
1. Удалить из конца все элементы <= текущего.
2. Добавить индекс текущего элемента в конец.
3. Удалить элемент спереди, если он вне окна.
4. Элемент в начале очереди — максимум текущего окна.
```

### Монотонно возрастающая очередь (для поиска минимума):

```text
1. Удалить из конца все элементы >= текущего.
2. Добавить индекс текущего элемента в конец.
3. Удалить элемент спереди, если он вне окна.
4. Элемент в начале очереди — минимум текущего окна.
```

## 🧪 Пример на Python
### Максимум в скользящем окне
```python
from collections import deque

def max_sliding_window(nums, k):
    if not nums or k == 0:
        return []

    dq = deque()
    result = []

    for i in range(len(nums)):
        if dq and dq[0] <= i - k:
            dq.popleft()

        while dq and nums[dq[-1]] <= nums[i]:
            dq.pop()

        dq.append(i)

        if i >= k - 1:
            result.append(nums[dq[0]])

    return result

# Примеры
print(max_sliding_window([1, 3, -1, -3, 5, 3, 6, 7], 3))  # [3, 3, 5, 5, 6, 7]
print(max_sliding_window([1], 1))                        # [1]
print(max_sliding_window([9, 11], 2))                    # [11]
print(max_sliding_window([7, 2, 4], 2))                  # [7, 4]
```

### Минимум в скользящем окне
```python 
def min_sliding_window(nums, k):
    if not nums or k == 0:
        return []

    dq = deque()
    result = []

    for i in range(len(nums)):
        if dq and dq[0] <= i - k:
            dq.popleft()

        while dq and nums[dq[-1]] >= nums[i]:
            dq.pop()

        dq.append(i)

        if i >= k - 1:
            result.append(nums[dq[0]])

    return result

# Пример
print(min_sliding_window([1, 3, -1, -3, 5, 3, 6, 7], 3))  # [-1, -3, -3, -3, 3, 3]
```

## ⏱️ Сложность
|Операция        | Временная сложность | Пространственная сложность |
|:---------------|:--------------------|:---------------------------|
|Общая работа	 | O(N)	               | O(K)                       |
|Операции с deque| O(1) на операцию    | До K элементов             |

* Каждый элемент добавляется и удаляется из deque не более одного раза.

* Все операции с deque выполняются за константное время.

---

## ⚠️ Особенности

- **Храните индексы, а не значения** — проще проверять, входит ли элемент в окно.
- **Строгая vs нестрогая монотонность** — выбирайте `<`, `<=`, `>`, `>=` в зависимости от требований к дубликатам.
- **Проверка на пустоту** — всегда проверяйте, не пуст `deque`, перед обращением к `dq[0]`.
- **Сбор результатов** — только после полного формирования окна (`i >= k - 1`).

## 🧭 Применение

- **Максимум/минимум в скользящем окне** — главное применение.
- **Следующий больший/меньший элемент** — аналогичная идея с использованием стека.
- **Оптимизация динамического программирования** — снижение сложности с `O(N²)` до `O(N)`.
- **Запросы на диапазон подмассивов** — при зависимости от min/max.
- **Планирование и жадные алгоритмы** — выбор оптимальных кандидатов на лету.
