# 🎯 K Найкращих Елементів (Підхід: Купа + Мапа)

## 📌 Опис Проблеми

Задача "K найкращих елементів" вимагає від нас знайти `k` **найбільших** (або **найменших**) елементів із заданого списку, масиву або потоку даних. Поширеним варіантом є також пошук `k` **найбільш часто зустрічаючихся** елементів.

**Приклади:**
* Дано `[3, 2, 1, 5, 6, 4]` та `k = 2`, знайти 2 найбільші елементи. Вихід: `[5, 6]` (порядок зазвичай не має значення).
* Дано `["apple", "banana", "apple", "orange", "banana", "apple"]` та `k = 2`, знайти 2 найбільш часто зустрічаючі слова. Вихід: `["apple", "banana"]` (якщо кількість збігів однакова, може бути вказаний алфавітний або початковий порядок).

## 💡 Основна Ідея та Підхід

Наївний підхід полягав би в сортуванні всього масиву, а потім виборі перших `k` елементів. Це зайняло б $O(N \log N)$ часу. Ми можемо досягти кращих результатів, використовуючи **купу**.

Основна ідея полягає в підтримці купи розміру `k`, яка в будь-який момент містить `k` "найкращих" елементів, що були зустрінуті до цього.

### Для K Найбільших Елементів: Використовуйте Мін-Купу

* Ми хочемо знайти `k` найбільших елементів.
* Ми використовуємо **Мін-Купу (Min-Heap)** розміру `k`.
* Чому Мін-Купа? Тому що найменший елемент у нашій купі (її корінь) є `k`-м за величиною елементом, який ми знайшли до цього. Якщо ми зустрічаємо новий елемент, який *більший* за цей `k`-й за величиною елемент, це означає, що новий елемент є одним зі справжніх `k` найкращих елементів, а поточний `k`-й за величиною елемент можна відкинути.

### Для K Найбільш Часто Зустрічаючихся Елементів: Використовуйте Мапу + Мін-Купу

Це двохетапний процес:

1.  **Підрахунок Частот:** Використовуйте **Мапу (Хеш-таблицю/Словник)** для підрахунку кількості входжень кожного унікального елемента у вхідних даних. Це перетворює задачу з "елементів" на "пари елемент-частота".
2.  **Пошук K Найвищих Частот:** Тепер застосуйте логіку "K найбільших елементів" до цих пар `(елемент, частота)`. Ми будемо використовувати **Мін-Купу**, яка пріоритезує за частотою. Якщо частоти збігаються, може бути вказаний вторинний критерій сортування (наприклад, алфавітний порядок елемента).

## 📝 Кроки Алгоритму (Детально)

### Випадок 1: Пошук K Найбільших Чисел

1.  **Ініціалізація Мін-Купи:** Створіть порожню мін-купу (пріоритетну чергу).
2.  **Ітерація по Вхідним Даним:** Для кожного `числа` у вхідному масиві/списку:
    * Якщо `розмір_мін_купи < k`:
        * `мін_купа.вставити(число)` (Додати число до купи).
    * Інакше, якщо `число > мін_купа.переглянути_найменше()`:
        * `мін_купа.видобути_найменше()` (Видалити найменший елемент з купи).
        * `мін_купа.вставити(число)` (Додати поточне `число` до купи).
    * Інакше (`число <= мін_купа.переглянути_найменше()`):
        * Нічого не робити. Це число не входить до `k` найбільших, зустрінутих до цього.
3.  **Результат:** Після ітерації по всіх числах, `мін_купа` буде містити `k` найбільших елементів. Потім ви можете витягти їх по одному (`мін_купа.видобути_найменше()`), якщо вам потрібен відсортований порядок, або просто повернути вміст купи як список.

### Випадок 2: Пошук K Найбільш Часто Зустрічаючихся Елементів

1.  **Підрахунок Частот:**
    * Створіть `мапу_частот` (наприклад, `collections.Counter` або словник у Python).
    * Проітеруйте вхідні дані та заповніть `мапу_частот`, де ключі — це елементи, а значення — їх частоти.
2.  **Ініціалізація Мін-Купи Кортежів:** Створіть порожню мін-купу. Ця купа буде зберігати кортежі `(частота, елемент)`. Купа буде пріоритезувати за першим елементом кортежу (частотою), тому вона ефективно буде мін-купою частот.
3.  **Ітерація по `мапі_частот`:** Для кожної пари `(елемент, частота)` у `мапі_частот`:
    * Якщо `розмір_мін_купи < k`:
        * `мін_купа.вставити((частота, елемент))`
    * Інакше, якщо `частота > мін_купа.переглянути_найменше().частота`: (Порівняти поточну частоту з частотою найменшого елемента в купі)
        * `мін_купа.видобути_найменше()`
        * `мін_купа.вставити((частота, елемент))`
    * Інакше (`частота <= мін_купа.переглянути_найменше()`):
        * Нічого не робити.
4.  **Результат:** Після ітерації по всіх парах частот, `мін_купа` буде містити `k` кортежів, де елементи відповідають `k` найбільш часто зустрічаючим елементам. Витягніть елементи (наприклад, другий елемент кожного кортежу), щоб отримати ваш результат.

## ⏱️ Аналіз Складності

* **Часова Складність:**
    * **Підрахунок Частот (якщо застосовується):** $O(N)$ для ітерації по всіх `N` елементах для побудови мапи.
    * **Операції з Купою:** Для кожного з `M` унікальних елементів (або `N` елементів у першому випадку) ми виконуємо щонайбільше одну операцію вставки/видалення з купи. Операція з купою займає $O(\log K)$ часу, оскільки максимальний розмір купи дорівнює `K`.
    * **Загальний Час:** $O(N + M \log K)$ при підрахунку частот (або $O(N \log K)$, якщо підрахунок частот не потрібен). Оскільки $M \le N$, це спрощується до **$O(N \log K)$**. Це ефективніше, ніж сортування за $O(N \log N)$, коли `K` значно менше за `N`.

* **Просторова Складність:**
    * **Мапа Частот (якщо застосовується):** $O(M)$, де `M` — кількість унікальних елементів. У найгіршому випадку $M=N$, отже $O(N)$.
    * **Купа:** $O(K)$, оскільки вона зберігає щонайбільше `K` елементів.
    * **Загальний Простір:** $O(M + K)$ (або $O(N + K)$ у найгіршому випадку для частот), що спрощується до **$O(N)$** у найгіршому випадку (всі унікальні елементи) або **$O(K)$**, якщо елементи не унікальні (для пошуку лише найбільших чисел). Точніше, це $O(\text{кількість унікальних елементів} + K)$.

## ⚠️ Варіації та Міркування

* **Використання Макс-Купи:** Ви *могли б* вставити всі `N` елементів у Макс-Купу ($O(N \log N)$), а потім `k` разів видобути елементи ($O(K \log N)$). Загалом $O(N \log N + K \log N) = O(N \log N)$. Це, як правило, менш ефективно, ніж підхід з мін-купою $O(N \log K)$, коли $K \ll N$.
* **Вирішення Суперечностей (Ties):** Для задач "найбільш часто зустрічаючихся" (most frequent), якщо два елементи мають однакову частоту і вам потрібно вибрати `k`, умова задачі зазвичай вказує правило вирішення суперечностей (наприклад, лексикографічний порядок). Ви б врахували це, додавши критерії вирішення суперечностей до вашого кортежу для купи, часто як від'ємне значення, якщо це має бути поведінка мін-купи для вирішення суперечностей, або просто безпосередньо для поведінки макс-купи.
    * Наприклад, для `k` найбільш частих, де суперечності вирішуються за алфавітом: `(частота, рядок_елемента)` для мін-купи (за замовчуванням Python працює, оскільки менші рядки йдуть раніше). Якщо вам потрібен зворотний алфавітний порядок, це може бути `(частота, -хеш(рядок_елемента))` або власний компаратор.
* **Мале `k` проти Великого `k`:** Підхід на основі купи особливо ефективний, коли `k` невелике відносно `N`. Якщо `k` близьке до `N`, сортування ($O(N \log N)$) може бути конкурентним або навіть простішим у реалізації, залежно від константних факторів.

## 🧪 Приклади на Python

### Приклад 1: K Найбільших Чисел

```python
import heapq

def find_top_k_largest(nums, k):
    if k == 0:
        return []

    # Створюємо мін-купу
    min_heap = []

    for num in nums:
        if len(min_heap) < k:
            # Якщо розмір купи менший за k, просто додаємо число
            heapq.heappush(min_heap, num)
        elif num > min_heap[0]: # min_heap[0] - це найменший елемент у мін-купі
            # Якщо поточне число більше за найменше в купі,
            # видаляємо найменше та додаємо поточне число
            heapq.heappop(min_heap)
            heapq.heappush(min_heap, num)

    # Мін-купа тепер містить k найбільших елементів
    return sorted(min_heap, reverse=True) # Сортуємо для відображення, інакше просто повертаємо min_heap

# Тестові випадки
print("--- K Найбільших Чисел ---")
nums1 = [3, 2, 1, 5, 6, 4]
k1 = 2
print(f"Числа: {nums1}, k: {k1} -> Top {k1}: {find_top_k_largest(nums1, k1)}") # Очікується: [6, 5]

nums2 = [7, 10, 4, 3, 20, 15]
k2 = 3
print(f"Числа: {nums2}, k: {k2} -> Top {k2}: {find_top_k_largest(nums2, k2)}") # Очікується: [20, 15, 10]

nums3 = [1, 2, 3, 4, 5]
k3 = 5
print(f"Числа: {nums3}, k: {k3} -> Top {k3}: {find_top_k_largest(nums3, k3)}") # Очікується: [5, 4, 3, 2, 1]

nums4 = [10, 10, 10, 9, 9, 8]
k4 = 2
print(f"Числа: {nums4}, k: {k4} -> Top {k4}: {find_top_k_largest(nums4, k4)}") # Очікується: [10, 10]
```

### Приклад 2: K Найбільш Часто Зустрічаючихся Елементів (Купа + Мапа)

```python
import heapq
from collections import Counter

def find_top_k_frequent(words, k):
    if k == 0:
        return []

    # Крок 1: Підраховуємо частоти за допомогою хеш-мапи (Counter)
    frequency_map = Counter(words)
    print(f"Мапа частот: {frequency_map}")

    # Крок 2: Використовуємо мін-купу для відстеження k найкращих елементів за частотою
    # Зберігаємо кортежі (частота, слово). heapq у Python - це мін-купа,
    # тому вона природно впорядковуватиме за частотою (перший елемент кортежу).
    min_heap = []

    for word, freq in frequency_map.items():
        if len(min_heap) < k:
            heapq.heappush(min_heap, (freq, word))
        elif freq > min_heap[0][0]: # Порівнюємо поточну частоту з найменшою частотою в купі
            heapq.heappop(min_heap)
            heapq.heappush(min_heap, (freq, word))
        # Необов'язково: Обробка суперечностей на основі вторинних критеріїв (наприклад, алфавітний порядок, якщо частота однакова)
        # elif freq == min_heap[0][0] and word < min_heap[0][1]: # Приклад для вирішення суперечностей: якщо поточне слово менше за алфавітом
        #     heapq.heappop(min_heap)
        #     heapq.heappush(min_heap, (freq, word))


    # Крок 3: Витягуємо елементи з купи
    # Купа містить кортежі (freq, word). Нам потрібні лише слова.
    result = [item[1] for item in min_heap]

    # Необов'язково: Сортуємо результат для послідовного виводу, якщо порядок має значення (наприклад, за спаданням частоти, потім за зростанням слова для суперечностей).
    # Зазвичай у задачі вказується порядок. Якщо ні, будь-який порядок k найкращих є прийнятним.
    # Тут ми сортуємо за спаданням частоти, потім за зростанням слова для суперечностей.
    result.sort(key=lambda word: (-frequency_map[word], word))

    return result

# Тестові випадки
print("\n--- K Найбільш Часто Зустрічаючихся Елементів ---")
words1 = ["i", "love", "leetcode", "i", "love", "coding"]
k1 = 2
print(f"Слова: {words1}, k: {k1} -> Топ {k1}: {find_top_k_frequent(words1, k1)}")
# Очікується: ["i", "love"] (або ["love", "i"] залежно від обробки суперечностей/внутрішнього порядку купи)

words2 = ["the", "day", "is", "sunny", "the", "the", "the", "sunny", "is", "is"]
k2 = 3
print(f"Слова: {words2}, k: {k2} -> Топ {k2}: {find_top_k_frequent(words2, k2)}")
# Очікується: ["the", "is", "sunny"]

words3 = ["a", "b", "c", "a", "b", "a"]
k3 = 1
print(f"Слова: {words3}, k: {k3} -> Топ {k3}: {find_top_k_frequent(words3, k3)}")
# Очікується: ["a"]
```
