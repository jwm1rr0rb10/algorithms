# 📐 Расширенный алгоритм Евклида

## 📌 Описание задачи

**Расширенный алгоритм Евклида** — это расширение классического алгоритма Евклида для нахождения НОД двух чисел `a` и `b`. Помимо самого `gcd(a, b)`, он также находит такие целые `x` и `y`, что:

- `a·x + b·y = gcd(a, b)`


Это называется **тождеством Безу**.

---

## 💡 Идея и подход

Алгоритм работает рекурсивно:

- Базовый случай:  
  Если `a = 0`, то `gcd(a, b) = b`, и `x = 0`, `y = 1`.

- Рекурсивный случай:  
  Вызываем `extended_gcd(b % a, a)` и восстанавливаем `x` и `y` через обратный ход рекурсии.

Применяется для:
- Нахождения обратного по модулю: `a⁻¹ mod m`, если `gcd(a, m) = 1`
- Решения линейных диофантовых уравнений
- Китайской теоремы об остатках (если модули не взаимно просты)

---

## 🧪 Пример на Python

```python
def extended_gcd(a, b):
    if a == 0:
        return b, 0, 1
    gcd, x1, y1 = extended_gcd(b % a, a)
    x = y1 - (b // a) * x1
    y = x1
    return gcd, x, y

# Пример: решить 30x + 20y = gcd(30, 20)
g, x, y = extended_gcd(30, 20)
print(f"gcd = {g}, x = {x}, y = {y}")  # Вывод: gcd = 10, x = 1, y = -1
```

---

## ⏱️ Сложность
- Время: O(log max(a, b))
- Память: O(log max(a, b)) из-за рекурсии

## 🧭 Применение
- Обратный по модулю: a⁻¹ mod m = x, если a·x ≡ 1 mod m
- Решение уравнений вида ax + by = c
- Китайская теорема об остатках (если модули не взаимно просты)
- Генерация ключей RSA (обратный элемент по модулю φ(n))
