# 🧮 Обратный элемент по модулю (Modular Inverse)

## 📌 Описание Проблемы

- Обратный элемент по модулю для целого числа a по модулю m — это целое число x такое, что:
```text
(a⋅x)≡1(modm)
```

- Обратный элемент по модулю существует тогда и только тогда, когда a и m взаимно просты (то есть их наибольший общий делитель (НОД) равен 1), т.е. textНОД`(a,m)=1`.

- Существует два основных метода для нахождения обратного элемента по модулю:

    - Расширенный алгоритм Евклида `(Extended Euclidean Algorithm)`: Этот метод универсален и работает для любого `m`.

    - Малая теорема Ферма `(Fermat's Little Theorem)`: Этот метод применим только в том случае, если `m` является простым числом.

 ---

## 💡 Идея и Подход

**1. Расширенный алгоритм Евклида**

Расширенный алгоритм Евклида находит целые числа x и y такие, что:
```text
a⋅x+m⋅y=НОД(a,m)
```

Если `$ \text{НОД}(a, m) = 1 $`, то уравнение принимает вид:
```text
a⋅x+m⋅y=1
```

Взяв это уравнение по модулю `m`:
```text
$$(a \cdot x + m \cdot y) \pmod{m}
\equiv 1 \pmod{m}$$$$(a \cdot x) \pmod{m} + 
(m \cdot y) \pmod{m} \equiv 1 \pmod{m}$$$$(a \cdot x) \pmod{m} +
 0 \equiv 1 \pmod{m}$$(a⋅x)(modm)≡1(modm)
```

Таким образом, `x`, найденный Расширенным алгоритмом Евклида, является обратным элементом a по модулю `m`. Если `x` отрицательное, мы можем прибавить к нему `m`, пока оно не станет положительным `(x=(x)`.

**2. Малая теорема Ферма (когда `m` — простое число)**

Малая теорема Ферма утверждает, что если `p` — простое число, то для любого целого числа a, не делящегося на `p`:
```text
ap−1≡1(modp)
```

Умножив обе стороны на `a−1`:
`ap−2≡a−1(modp)`

Следовательно, обратный элемент a по модулю простого числа `p` равен `ap−2pmodp`, что может быть эффективно вычислено с помощью модульного возведения в степень (бинарного возведения в степень).

## 🧪 Пример на Python
```python
def extended_gcd(a, b):
    if a == 0:
        return b, 0, 1
    gcd, x1, y1 = extended_gcd(b % a, a)
    x = y1 - (b // a) * x1
    y = x1
    return gcd, x, y

def modular_inverse_extended_euclidean(a, m):
    gcd, x, y = extended_gcd(a, m)
    if gcd != 1:
        return None  # Обратный элемент по модулю не существует
    else:
        return (x % m + m) % m # Убедимся, что результат положительный

def power(base, exp, mod):
    res = 1
    base %= mod
    while exp > 0:
        if exp % 2 == 1:
            res = (res * base) % mod
        base = (base * base) % mod
        exp //= 2
    return res

def modular_inverse_fermat_little_theorem(a, m):
    # Работает только если m - простое число
    # и a не является кратным m
    if m <= 1:
        return None # Модуль должен быть простым числом > 1
    # Для простоты здесь не добавлена проверка на простоту m, но на практике это критически важно.
    # В реальном сценарии вы бы добавили проверку m на простоту.
    
    # Если a кратно m, обратный элемент не существует
    if a % m == 0:
        return None 
    
    return power(a, m - 2, m)

# Примеры использования

# Использование Расширенного алгоритма Евклида
a = 3
m = 11 # 11 - простое, но этот метод работает для любого m, где НОД(a,m)=1
inv_ee = modular_inverse_extended_euclidean(a, m)
print(f"Обратный элемент {a} по модулю {m} (Расширенный Евклид): {inv_ee}") # Вывод: 4 (так как 3*4=12, 12%11=1)

a = 7
m = 26 # m не простое, поэтому Малая теорема Ферма не применима напрямую
inv_ee_non_prime = modular_inverse_extended_euclidean(a, m)
print(f"Обратный элемент {a} по модулю {m} (Расширенный Евклид): {inv_ee_non_prime}") # Вывод: 15 (так как 7*15=105, 105%26=1)

# Использование Малой теоремы Ферма (m должно быть простым)
a = 3
m_prime = 11
inv_flt = modular_inverse_fermat_little_theorem(a, m_prime)
print(f"Обратный элемент {a} по модулю {m_prime} (Малая теорема Ферма): {inv_flt}") # Вывод: 4

a = 5
m_prime_2 = 13
inv_flt_2 = modular_inverse_fermat_little_theorem(a, m_prime_2)
print(f"Обратный элемент {a} по модулю {m_prime_2} (Малая теорема Ферма): {inv_flt_2}") # Вывод: 8 (так как 5*8=40, 40%13=1)
```

## ⏱️ Сложность

1. Расширенный алгоритм Евклида

    `Время`: `O(log(min(a,m)))` - Аналогично вычислению НОД.

    `Пространство`: `O(log(min(a,m)))` для рекурсивных вызовов, или `O(1)` для итеративной версии.

2. Малая теорема Ферма (с модульным возведением в степень)

    `Время`: `O(logm)` - Из-за модульного возведения в степень.

    `Пространство`: `O(1)`

## 🧭 Применения

- `Криптография`: Важен в криптосистемах с открытым ключом, таких как RSA, для генерации ключей и расшифровки.

- `Модульная арифметика`: Решение линейных сравнений вида axequivbpmodm.

- `Теория чисел`: Различные задачи, связанные с делением по модулю. (Деление на a по модулю m эквивалентно умножению на обратный элемент a по модулю m).

- `Комбинаторика`: Вычисление "сочетаний из n по k" по модулю простого числа, где требуется деление на факториалы.

---

## ⚠️ Внимание
- `Существование`: Обратный элемент `a−1pmodm` существует тогда и только тогда, когда `textНОД(a,m)=1`. Всегда проверяйте это условие.

- `Ограничение Малой теоремы Ферма`: Эта теорема применима только тогда, когда модуль m является простым числом. Для составных модулей необходимо использовать Расширенный алгоритм Евклида.

- `Отрицательные результаты`: Расширенный алгоритм Евклида может вернуть отрицательное `x`. Не забудьте преобразовать его в положительный эквивалент, прибавив `m (например, x=(x)`.
