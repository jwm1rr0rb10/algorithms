# ⚡ Тест на простоту: Алгоритм Миллера-Рабина

## 📌 Описание задачи

**Тест Миллера–Рабина** — это вероятностный алгоритм, предназначенный для определения, является ли заданное число `n` простым или составным. В отличие от детерминированных тестов на простоту (которые медленнее для очень больших чисел), Миллер–Рабин предоставляет высокую вероятность корректности, что делает его крайне полезным в таких областях, как криптография.

Если `n` — простое число, оно всегда пройдёт проверку. Если составное — вероятность ложноположительного результата мала. Повторное выполнение алгоритма `k` раз значительно снижает вероятность ошибки.

---

## 💡 Идея и подход

Алгоритм опирается на два ключевых свойства простых чисел:

1. **Малая теорема Ферма**: Если `p` — простое, то для любого целого `a`, где `1 < a < p`, выполняется: `a^(p−1) ≡ 1 (mod p)`.

2. **Квадратичные вычеты (корни из 1 по модулю p)**: Если `p` — нечётное простое, то единственные решения уравнения `x^2 ≡ 1 (mod p)` — это `x ≡ 1 (mod p)` и `x ≡ -1 (mod p)`.

### Этапы алгоритма:

1. **Обработка тривиальных случаев:**

   - Если `n < 2`, вернуть `False`.
   - Если `n == 2` или `n == 3`, вернуть `True`.
   - Если `n` чётное и `n > 2`, вернуть `False`.

2. **Представить **``** в виде **``**, где **``** нечётное:**

   - Повторно делим `n - 1` на `2`, пока не получим нечётное `d`.

3. **Повторить тест **``** раз:**

   - Случайным образом выбрать `a` из диапазона `[2, n-2]`.
   - Вычислить `x = a^d % n`.
   - Если `x == 1` или `x == n - 1`, продолжить к следующей итерации.
   - Иначе, повторно возводим `x` в квадрат до `s - 1` раз:
     - Если на каком-либо этапе `x == n - 1`, продолжить к следующей итерации.
     - Если `x == 1`, вернуть `False` (точно составное).
   - Если ни одно условие не выполнено, вернуть `False`.

4. Если после всех `k` раундов `n` ни разу не было определено как составное, вернуть `True` — число **вероятно простое**.

---

## 🤔 Почему это работает

Если `n` — простое:

- Тогда по малой теореме Ферма: `a^(n−1) ≡ 1 (mod n)`.
- Последовательность: `a^d, a^{2d}, a^{4d}, ..., a^{2^s d}` обязательно должна быть либо:
  - Равна `1` на первом шаге, либо
  - Достичь `-1 (mod n)` перед тем, как достичь `1`.

Если это не так — `n` не является простым.

## 🧪 Пример на Python

```python
import random

def power(base, exp, mod):
    res = 1
    base %= mod
    while exp > 0:
        if exp % 2 == 1:
            res = (res * base) % mod
        base = (base * base) % mod
        exp //= 2
    return res

def miller_rabin_test(d, n):
    a = random.randint(2, n - 2)
    x = power(a, d, n)
    if x == 1 or x == n - 1:
        return True
    while d != n - 1:
        x = (x * x) % n
        d *= 2
        if x == n - 1:
            return True
        if x == 1:
            return False
    return False

def is_prime_miller_rabin(n, k=5):
    if n <= 1: return False
    if n == 2 or n == 3: return True
    if n % 2 == 0: return False
    s = 0
    d = n - 1
    while d % 2 == 0:
        d //= 2
        s += 1
    for _ in range(k):
        if not miller_rabin_test(d, n):
            return False
    return True

# Примеры
print(is_prime_miller_rabin(2)) # True
print(is_prime_miller_rabin(13)) # True
print(is_prime_miller_rabin(15)) # False
print(is_prime_miller_rabin(561)) # False (число Кармайкла)
print(is_prime_miller_rabin(999999937, k=10)) # True
print(is_prime_miller_rabin(1000000007, k=10)) # True
```

---

## ⏱️ Сложность

- **Время:** `O(k * log^3(n))`

  - `k` — количество итераций (тестов).
  - Каждая итерация включает модульное возведение в степень за `O(log n)` шагов.
  - Каждое умножение выполняется за `O(log^2 n)`.

- **Память:** `O(log n)`

  - Переменные хранят значения до `n`, значит используются `log n` бит.

---

## ⚠️ Вероятностная природа

- Для составного числа вероятность пройти один тест ≤ 1/4.
- После `k` итераций вероятность ошибки ≤ (1/4)^k.
- В криптографии используют `k=40`, что даёт вероятность ошибки ≈ 10^-24.
- Для маленьких чисел (например, < 2^64) можно использовать фиксированные базы `a`, делая тест **детерминированным**.
