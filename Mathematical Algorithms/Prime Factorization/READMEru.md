# 🔍 Разложение на Простые Множители

## 📌 Описание Задачи

**Разложение на простые множители** — это процесс разбиения составного числа на его простые множители — простые числа, которые при перемножении дают исходное число. Например, простыми множителями числа 12 являются 2, 2 и 3 ($2 \times 2 \times 3 = 12$). Каждое составное число имеет уникальное разложение на простые множители (Основная теорема арифметики).

---

## 💡 Идея и Подход

Наиболее простой подход для разложения на простые множители включает метод пробного деления:

1.  **Деление на 2:** Последовательно делите число `n` на 2, пока оно остается делимым. Подсчитайте, сколько раз 2 было множителем.
2.  **Деление на нечетные числа:** Итерируйте по нечетным числам `i`, начиная с 3, до $\sqrt{n}$. Для каждого `i` последовательно делите `n` на `i`, пока оно делится. Подсчитайте, сколько раз `i` было множителем.
3.  **Обработка оставшегося числа:** Если после этих делений `n` все еще больше 1, это означает, что оставшееся `n` само является простым числом (и наибольшим простым множителем).

Этот метод работает, потому что любое составное число `n` должно иметь по крайней мере один простой множитель, меньший или равный $\sqrt{n}$. Если это не так, то все его простые множители должны быть больше $\sqrt{n}$. В этом случае произведение любых двух таких множителей было бы больше $n$, что является противоречием.

---

## 🧪 Пример на Python

```python
def prime_factorization(n):
    factors = {} # Используем словарь для хранения множителей и их количества

    # Обработка множителя 2
    while n % 2 == 0:
        factors[2] = factors.get(2, 0) + 1
        n //= 2

    # Обработка нечетных множителей от 3 до sqrt(n)
    i = 3
    while i * i <= n:
        while n % i == 0:
            factors[i] = factors.get(i, 0) + 1
            n //= i
        i += 2 # Проверяем только нечетные числа

    # Если n все еще больше 1, оно само должно быть простым множителем
    if n > 1:
        factors[n] = factors.get(n, 0) + 1

    return factors

# Пример
print(prime_factorization(12)) # Вывод: {2: 2, 3: 1}
print(prime_factorization(100)) # Вывод: {2: 2, 5: 2}
print(prime_factorization(17)) # Вывод: {17: 1}
print(prime_factorization(999999937)) # Вывод: {999999937: 1} (простое число)
print(prime_factorization(720)) # Вывод: {2: 4, 3: 2, 5: 1}
```

## ⏱️ Сложность

* **Временная сложность:** $O(\sqrt{n})$ в худшем случае (когда $n$ является простым числом или произведением двух больших простых чисел). Это связано с тем, что цикл итерирует до $\sqrt{n}$.
* **Пространственная сложность:** $O(\log n)$ или $O(\text{количество различных простых множителей})$ для хранения множителей. В худшем случае число может иметь много маленьких простых множителей (например, $2^k$), или несколько больших различных. Количество различных простых множителей числа $n$ обычно значительно меньше, чем $O(\log n)$.

---

## ⚠️ Особенности для больших чисел

Для очень больших чисел (например, сотни цифр) метод пробного деления со сложностью $O(\sqrt{n})$ становится слишком медленным. В таких случаях используются специализированные алгоритмы:

* **Алгоритм Полларда "Ро"**: Более эффективен, чем пробное деление, для чисел с относительно небольшими простыми множителями, но не гарантирует быстрого нахождения множителей.
* **Алгоритм Полларда p-1**: Эффективен, если $n$ имеет простой множитель $p$ такой, что $p-1$ является произведением малых простых чисел.
* **Метод эллиптических кривых (ECM)**: Высокоэффективный алгоритм общего назначения для нахождения относительно небольших множителей больших чисел.
* **Общий метод решета числового поля (GNFS)**: Самый быстрый известный алгоритм для факторизации очень больших чисел (например, более 100 цифр), часто используемый в криптографии.

## 🧭 Применение

* **Криптография:** Критически важен для безопасности таких алгоритмов, как RSA, который основывается на сложности факторизации больших чисел.
* **Теория чисел:** Фундаментальная операция для многих задач, включая нахождение НОД/НОК, определение совершенных чисел и решение диофантовых уравнений.
* **Сжатие данных:** Некоторые алгоритмы сжатия используют свойства, связанные с простой факторизацией.
* **Генерация простых чисел:** Хотя прямая факторизация не используется для генерации простых чисел, понимание их распределения жизненно важно.
* **Упрощение дробей:** Приведение дробей к простейшему виду включает деление числителя и знаменателя на их общие простые множители.
