# Обзор алгоритмов "Два указателя" и "Скользящее окно"

## Введение

Алгоритмы, использующие техники **двух указателей** и **скользящего окна**, — мощные инструменты для эффективного решения задач на массивах и строках. Они позволяют обрабатывать данные за линейное или почти линейное время, часто с минимальными дополнительными затратами памяти.

---

## Быстрое сравнение задач

| Задача                                       | Время     | Память   | Подход                            |
|-----------------------------------------------|-----------|----------|-----------------------------------|
| Сравнение строк с backspace                   | O(n)      | O(1)     | Два указателя                     |
| Контейнер с наибольшей площадью               | O(n)      | O(1)     | Два указателя                     |
| Все анаграммы в строке                        | O(n)      | O(1)/O(k)| Скользящее окно                   |
| Четыре суммы                                  | O(n²/n³)  | O(n)     | Два указателя + хэш               |
| Замена символов до максимальной повторяемости  | O(n)      | O(1)     | Скользящее окно                   |
| Самая длинная подстрока без повторяющихся симв.| O(n)     | O(n)     | Скользящее окно + множество       |
| Максимальная сумма подмассива (Кадане)        | O(n)      | O(1)     | Скользящее окно                   |
| Минимальная длина подмассива с суммой         | O(n)      | O(1)     | Скользящее окно                   |
| Минимальное окно в строке                     | O(n)      | O(n)     | Скользящее окно + Map             |
| Переместить нули                              | O(n)      | O(1)     | Два указателя                     |
| Удалить дубликаты в отсортированном массиве   | O(n)      | O(1)     | Два указателя                     |
| Сортировка цветов (голландский флаг)          | O(n)      | O(1)     | Два указателя                     |
| Три суммы                                     | O(n²/n³)  | O(n)     | Два указателя + сортировка        |
| Ловушка для воды                              | O(n)      | O(1)     | Два указателя                     |
| Две суммы                                     | O(n²/n³)  | O(n)     | Хэш-таблица / два указателя       |
| **Максимальная сумма окна (пример ниже)**     | O(n)      | O(1)     | Скользящее окно                   |

## Пример скользящего окна

**Задача:** Найти максимальную сумму подмассива длины `k` в массиве.

```python
def max_sum_subarray(arr, k):
    window_sum = sum(arr[:k])
    max_sum = window_sum

    for i in range(k, len(arr)):
        window_sum += arr[i] - arr[i - k]
        max_sum = max(max_sum, window_sum)

    return max_sum
```

- **Временная сложность:** O(n)
- **Пространственная сложность:** O(1)

## Когда и где применять

### Два указателя

- **Лучше всего подходит для:**  
  - Задач с поиском пар/групп с определёнными свойствами.
  - Обработки отсортированных массивов (удаление дубликатов, сортировка, задачи на сумму).
  - Сжатия или расширения интервалов без полного перебора всех комбинаций.
  - Задач, где желательна минимальная дополнительная память (обычно O(1)).

- **Типичные примеры:**  
  - Контейнер с наибольшей площадью, перемещение нулей, удаление дубликатов, сортировка цветов, три/четыре суммы, ловушка для воды.

- **Ограничения:**  
  - Менее эффективно для неотсортированных массивов без вспомогательных структур.
  - Иногда требуется предварительная сортировка (дополнительные затраты времени).
  - Не подходит для задач, где порядок элементов критичен и нельзя двигаться с обоих концов.

### Скользящее окно

- **Лучше всего подходит для:**  
  - Поиска подстрок/подмассивов с определёнными свойствами (максимальная длина, сумма, число уникальных символов и т.д.).
  - Задач с динамическим окном: "расширяем вправо", "сужаем слева" с сохранением инвариантов.
  - Обработки больших данных за линейное время.

- **Типичные примеры:**  
  - Самая длинная подстрока без повторяющихся символов, минимальное окно в строке, максимальная сумма подмассива, минимальная длина подмассива с суммой, все анаграммы, максимальная сумма окна.

- **Ограничения:**  
  - Неприменимо, если задачу нельзя свести к окну фиксированного или переменного размера.
  - Для сложных правил окна (например, подсчёт вхождений) нужны дополнительные структуры (массивы/словари).
  - Может требовать больше памяти для хранения частот.

---

## Когда НЕ использовать

- Если требуется полный перебор всех комбинаций без оптимизации через окно или указатели.
- Если данные очень неструктурированы и сложно определить инварианты для движения окна.
- Если задача требует вложенных окон или нескольких проходов по одним и тем же данным (тогда лучше DP или жадные методы).

---

## Ограничения подходов

- **Два указателя** не всегда эффективен, если нужно хранить дополнительную информацию о посещённых элементах или структура данных не подходит.
- **Скользящее окно** неэффективно, если нельзя быстро обновлять состояние окна (например, сложные подсчёты или дорогие операции).

---

## Общие советы по выбору подхода

1. **Анализируйте ограничения задачи** — если требуется линейное время и есть "движущееся окно" или "два конца", скорее всего, можно применить одну из техник.
2. **Проверьте, можно ли разбить задачу на подзадачи с помощью окна или указателей** — например, найти границы, где выполняется инвариант.
3. **Рассмотрите сортировку данных для применения двух указателей, если это оправдано.**
4. **Не бойтесь комбинировать подходы:** скользящее окно + хэш-таблица, два указателя + сортировка и т.д.

## Заключение
