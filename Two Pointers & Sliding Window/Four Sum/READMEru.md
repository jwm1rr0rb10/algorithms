# Четыре суммы — сортировка и два указателя

## Описание задачи

Дан массив `nums` из n целых чисел и целое число `target`. Необходимо найти все уникальные четверки `[a, b, c, d]`, такие что `a + b + c + d == target`.

Дубликаты четверок возвращать нельзя, порядок внутри четверки не важен.

**Пример:**  
Ввод: nums = [1, 0, -1, 0, -2, 2], target = 0  
Вывод:  
```
[
  [-2, -1, 1, 2],
  [-2, 0, 0, 2],
  [-1, 0, 0, 1]
]
```

---

## Идея и подход алгоритма

- Отсортировать массив для удобства поиска и устранения дубликатов.
- Использовать два вложенных цикла для выбора первых двух чисел (`i` и `j`).
- Для каждой пары применяем два указателя (`left` и `right`), чтобы найти такие пары, что сумма четырех чисел равна target.
- Двигаем указатели к центру, пропуская дубликаты, и сохраняем найденные четверки.
- Продолжаем до перебора всех уникальных четверок.

---

## Пример на Python

```python
def fourSum(nums, target):
    nums.sort()
    n = len(nums)
    result = []

    for i in range(n - 3):
        if i > 0 and nums[i] == nums[i-1]:
            continue  # пропуск дубликатов для i
        for j in range(i + 1, n - 2):
            if j > i + 1 and nums[j] == nums[j-1]:
                continue  # пропуск дубликатов для j
            left, right = j + 1, n - 1
            while left < right:
                curr_sum = nums[i] + nums[j] + nums[left] + nums[right]
                if curr_sum == target:
                    result.append([nums[i], nums[j], nums[left], nums[right]])
                    left += 1
                    right -= 1
                    while left < right and nums[left] == nums[left-1]:
                        left += 1
                    while left < right and nums[right] == nums[right+1]:
                        right -= 1
                elif curr_sum < target:
                    left += 1
                else:
                    right -= 1
    return result

# Пример использования:
print(fourSum([1, 0, -1, 0, -2, 2], 0))
# Выведет: [[-2, -1, 1, 2], [-2, 0, 0, 2], [-1, 0, 0, 1]]
```

---

## Анализ сложности

- **Время:** O(n³) — для каждой пары (i, j) два указателя просматривают оставшуюся часть (n² * n).
- **Память:** O(k), где k — число уникальных четверок (не считая сортировки входных данных).
