# 📐 Обзор алгоритмов геометрии

Алгоритмы геометрии необходимы для решения задач, связанных с формами, расстояниями, пересечениями и пространственными отношениями. Они широко применяются в вычислительной геометрии, компьютерной графике, робототехнике, ГИС и системах обнаружения столкновений.

Этот раздел содержит сводку ключевых геометрических алгоритмов, их временных и пространственных сложностей, а также рекомендации по применению.

---

## 📊 Таблица сложности алгоритмов

| Алгоритм                                 | Временная сложность | Пространственная сложность |
|:------------------------------------------|:----------------------|:-----------------------------|
| Площадь многоугольника                   | O(n)                 | O(1)                        |
| Пересечение окружностей                  | O(1)                 | O(1)                        |
| Ближайшая пара точек                     | O(n log n)           | O(n)                        |
| Выпуклая оболочка (алгоритм Грэхема)     | O(n log n)           | O(n)                        |
| Выпуклая оболочка (алгоритм Джарвиса)    | O(nh)                | O(1)                        |
| Пересечение прямых                       | O(1)                 | O(1)                        |
| Пересечение отрезков                     | O(1)                 | O(1)                        |
| Принадлежность точки многоугольнику (Ray Casting) | O(n)         | O(1)                        |
| Пересечение прямоугольников              | O(1)                 | O(1)                        |
| Наложение прямоугольников                | O(1)                 | O(1)                        |
| Вращающиеся штангенциркули (диаметр выпуклого многоугольника) | O(n) | O(1) |

---

## 🧭 Когда использовать эти алгоритмы

- **Обнаружение столкновений**:  
  Пересечение прямоугольников, отрезков и окружностей применяются в игровых движках, интерфейсах и робототехнике.

- **Анализ формы**:  
  Площадь многоугольника, выпуклая оболочка и вращающиеся штангенциркули полезны в компьютерном зрении и распознавании образов.

- **Геоинформационные системы (ГИС)**:  
  Проверка принадлежности точки многоугольнику и поиск ближайшей пары точек — основа для картографических и пространственных задач.

- **Оптимизация и планирование**:  
  Выпуклая оболочка и вращающиеся штангенциркули применяются для построения маршрутов, минимизации ограничивающих рамок и подгонки форм.

---

## ⚠️ На что обратить внимание

- Выпуклая оболочка (алгоритм Джарвиса) может работать медленно при h ≈ n (вплоть до O(n²)).
- Алгоритм ближайшей пары точек требует аккуратной реализации divide-and-conquer для сохранения O(n log n).
- Метод Ray Casting предполагает простой многоугольник — при самопересечениях возможны ошибки.
- Погрешности с плавающей точкой могут влиять на точность пересечений и ориентации — используйте устойчивые геометрические предикаты.

---

## 🛠️ Практические советы

- Предобрабатывайте входные данные: удаляйте дубликаты и сортируйте при необходимости (например, для выпуклой оболочки).
- Кэшируйте выпуклые оболочки для статичных объектов.
- Используйте пространственные структуры данных (QuadTree, KD-Tree) для масштабируемых или real-time задач.
- При возможности используйте целочисленные координаты для избежания ошибок округления.
