# Алгоритм Беллмана-Форда

**Алгоритм Беллмана-Форда** — это классический алгоритм для поиска кратчайших путей из одной вершины ко всем остальным в ориентированном взвешенном графе.  
В отличие от алгоритма Дейкстры, он корректно работает при наличии отрицательных весов рёбер и может обнаруживать отрицательные циклы.

---

## Когда применять

- В графе могут быть отрицательные веса рёбер.
- Нужно обнаруживать отрицательные циклы.
- Требуется простой и надёжный алгоритм для разреженных графов.

---

## Как работает алгоритм

1. **Инициализация:**  
   Задаём расстояние от источника до себя как 0, до остальных вершин — бесконечность.

2. **Релаксация:**  
   Повторяем V-1 раз (V — число вершин):  
   Для каждого ребра (u, v, w), если `distance[u] + w < distance[v]`, то обновляем `distance[v] = distance[u] + w`.

3. **Проверка на отрицательные циклы:**  
   Проходим по всем рёбрам ещё раз.  
   Если удаётся ещё уменьшить какое-либо расстояние — в графе есть отрицательный цикл.

---

## Сложность

| Время           | Память        |
|:---------------:|:-------------:|
| O(VE)           | O(V)          |

- **V** — число вершин  
- **E** — число рёбер

---

## Пример на Python

```python
def bellman_ford(V, edges, source):
    """
    V — количество вершин (от 0 до V-1)
    edges — список троек (u, v, w): ребро из u в v весом w
    source — стартовая вершина
    Возвращает: (список расстояний, есть_ли_отрицательный_цикл)
    """
    INF = float('inf')
    dist = [INF] * V
    dist[source] = 0

    # Релаксация рёбер V-1 раз
    for _ in range(V - 1):
        for u, v, w in edges:
            if dist[u] != INF and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w

    # Проверка на отрицательные циклы
    for u, v, w in edges:
        if dist[u] != INF and dist[u] + w < dist[v]:
            return dist, True  # Есть отрицательный цикл

    return dist, False

# Пример использования
V = 5
edges = [
    (0, 1, 6),
    (0, 2, 7),
    (1, 2, 8),
    (1, 3, 5),
    (1, 4, -4),
    (2, 3, -3),
    (2, 4, 9),
    (3, 1, -2),
    (4, 3, 7),
    (4, 0, 2)
]
source = 0
distances, has_negative_cycle = bellman_ford(V, edges, source)
print("Расстояния:", distances)
print("Есть ли отрицательный цикл?", has_negative_cycle)
```

---

## Применение

- Поиск кратчайших путей с отрицательными весами
- Поиск отрицательных циклов (арбитраж, долговые цепочки и пр.)
- Маршрутизация в сетях (например, протокол RIP)
- Валютный арбитраж
