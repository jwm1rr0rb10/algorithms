# Поиск в глубину (DFS, Depth-First Search)

## Что это такое

DFS (поиск в глубину) — алгоритм обхода или поиска в структурах данных, таких как деревья и графы. Он исследует каждую ветвь до конца прежде, чем вернуться назад (“откатиться”) и начать обходить другую ветвь.

---

## Как это работает

Представьте, что вы идёте по лабиринту: DFS — это как выбрать один путь и идти по нему до упора, пока не упрётесь в тупик или уже посещённую точку. После этого возвращаетесь к последнему месту выбора и пробуете другой неосмотренный путь.

---

## Формальное описание

1. Начните с выбранной вершины.
2. Посетите вершину и отметьте её как посещённую (чтобы не попасть в цикл).
3. Выберите одного из её непосещённых соседей.
4. Рекурсивно (или с помощью стека) примените DFS к этому соседу.
5. Если у текущей вершины нет непосещённых соседей — вернитесь к предыдущей (завершите рекурсивный вызов или извлеките вершину из стека) и продолжайте обход с неё.
6. Повторяйте до тех пор, пока не будут посещены все достижимые вершины или пока не найдена целевая (если ищем путь).

---

## Реализация

DFS можно реализовать:

* **Рекурсивно:** используется стек вызовов функций для имитации “глубины” обхода.
* **Вручную с помощью стека:** сами создаём стек, кладём стартовую вершину, пока стек не пуст — извлекаем вершину, посещаем, кладём в стек всех её непосещённых соседей.

---

## Сложность

* **Время:** $O(V + E)$, где V — количество вершин, E — количество рёбер. Каждая вершина и каждое ребро посещаются максимум один раз.
    * Для деревьев ($E = V - 1$) упрощается до O(V).
* **Память:** $O(V)$ в худшем случае — если граф “глубокий и узкий” (например, список), стек вызовов/стек может содержать до V вершин.

---

## Где применяется DFS на практике

1. **Поиск пути между двумя вершинами в лабиринте или сети**
    - В играх (поиск пути для бота/юнита).
    - Поиск файлов и папок в файловой системе (рекурсивный поиск).

2. **Обнаружение циклов в графе**
    - В графах зависимостей (например, менеджеры пакетов, системы сборки).

3. **Топологическая сортировка**
    - Планирование задач с зависимостями.

4. **Поиск сильносвязанных компонент в ориентированных графах**
    - Анализ социальных сетей, структуры ссылок, оптимизация компиляторов.

5. **Поиск мостов и точек сочленения (разрезов)**
    - Анализ надёжности сетей.

6. **Решение головоломок и задач перебора (backtracking)**
    - Судоку, N-Queens, генерация и решение лабиринтов.

## Пример на Go (рекурсивный DFS в графе)

```go
package main

import "fmt"

type Graph map[int][]int

func DFS(graph Graph, startNode int, visited map[int]bool) {
	visited[startNode] = true
	fmt.Printf("Посещена вершина: %d\n", startNode)

	neighbors, ok := graph[startNode]
	if !ok {
		return
	}

	for _, neighbor := range neighbors {
		if !visited[neighbor] {
			DFS(graph, neighbor, visited)
		}
	}
}

func main() {
	graph := Graph{
		0: {1, 2},
		1: {0, 3, 4},
		2: {0, 5},
		3: {1},
		4: {1},
		5: {2},
		6: {7},
		7: {6},
	}

	visited := make(map[int]bool)
	fmt.Println("Обход DFS с вершины 0:")
	DFS(graph, 0, visited)

	fmt.Println("\nОбход DFS для оставшихся непосещённых компонент:")
	for node := range graph {
		if !visited[node] {
			DFS(graph, node, visited)
		}
	}
}
```

---

## Пример на Python

```python
# Поиск в глубину (DFS)
graph = {
    0: [1, 2],
    1: [0, 3, 4],
    2: [0, 5],
    3: [1],
    4: [1],
    5: [2],
    6: [7],
    7: [6],
}

visited = set()

def dfs(visited, graph, node):
    visited.add(node)
    print(f"Посещена вершина: {node}")
    for neighbor in graph.get(node, []):
        if neighbor not in visited:
            dfs(visited, graph, neighbor)

print("Обход DFS с вершины 0:")
dfs(visited, graph, 0)

print("\nОбход DFS для оставшихся непосещённых компонент:")
for node in graph:
    if node not in visited:
        dfs(visited, graph, node)
```
