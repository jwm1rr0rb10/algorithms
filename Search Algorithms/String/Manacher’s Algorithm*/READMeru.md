# Алгоритм Манакера (Manacher’s Algorithm)

---

## Описание задачи

Дана строка. Найти все подстроки-палиндромы (палиндромные подстроки) за линейное время, а также определить длину самой длинной палиндромной подстроки.

---

## Подход

Алгоритм Манакера находит все палиндромные подстроки за O(n) времени.  
Идея:  
- "Расширять" палиндромы вокруг центра, используя информацию о уже найденных палиндромах.
- Для удобства обработать строку, вставив специальные символы (`#`) между буквами (и по краям) — так обрабатываются палиндромы как чётной, так и нечётной длины.

---

## Пример на Python

```python
def manacher(s):
    # Преобразуем строку: вставляем # между всеми символами и по краям
    t = '#' + '#'.join(s) + '#'
    n = len(t)
    p = [0] * n  # p[i] — радиус палиндрома с центром в i
    c = r = 0    # c — центр, r — правая граница самого правого палиндрома
    for i in range(n):
        mirr = 2 * c - i
        if i < r:
            p[i] = min(r - i, p[mirr])
        # Расширяем палиндром вокруг i
        a, b = i + 1 + p[i], i - 1 - p[i]
        while a < n and b >= 0 and t[a] == t[b]:
            p[i] += 1
            a += 1
            b -= 1
        # Обновляем центр и правую границу
        if i + p[i] > r:
            c, r = i, i + p[i]
    # Находим максимальный палиндром
    max_len = max(p)
    center_index = p.index(max_len)
    start = (center_index - max_len) // 2
    return s[start:start + max_len]

# Пример:
s = "babad"
print(manacher(s))  # Вывод: "bab" или "aba"
```

---

## Как это работает

- После специальной обработки строки палиндромы всегда нечётной длины.
- p[i] — максимальный радиус палиндрома с центром в позиции i.
- Используем симметрию относительно центра самого правого найденного палиндрома для ускорения поиска.

---

## Применение

- Поиск самой длинной палиндромной подстроки
- Быстрое (за O(n)) перечисление палиндромных подстрок
- Используется в задачах анализа строк, биоинформатике, сжатии данных и др.

---

## Когда использовать

- Когда требуется найти палиндромы (особенно — самый длинный) во входной строке очень быстро (O(n))

---

## Когда не стоит использовать

- Если требуется только проверить, является ли строка палиндромом (это быстрее за O(n))
- Если строка очень короткая — можно обойтись простым перебором

---

## Сложность

- **Время:** O(n)
- **Память:** O(n)
