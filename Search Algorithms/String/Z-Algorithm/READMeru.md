# Z-Алгоритм (Z-Algorithm)

---

## Описание задачи

Z-алгоритм — это эффективный способ для поиска подстрок и обработки строковых шаблонов.  
Он вычисляет массив Z для строки:  
Z[i] — длина наибольшего префикса строки, совпадающего с подстрокой, начинающейся с позиции i.

Применение:
- Поиск подстроки (pattern matching)
- Подсчёт повторяющихся подстрок
- Быстрое нахождение всех вхождений шаблона в тексте

---

## Подход

Z-алгоритм строит Z-массив за O(n) времени для строки длины n.  
Идея:  
- Вводим окно [l, r], в котором уже совпали символы.
- Для каждой позиции стараемся максимально расширить окно, используя ранее посчитанные значения.

---

## Пример на Python

```python
def z_algorithm(s):
    n = len(s)
    z = [0] * n
    l = r = 0
    for i in range(1, n):
        if i <= r:
            z[i] = min(r - i + 1, z[i - l])
        while i + z[i] < n and s[z[i]] == s[i + z[i]]:
            z[i] += 1
        if i + z[i] - 1 > r:
            l, r = i, i + z[i] - 1
    return z

# Пример:
s = "abacaba"
print(z_algorithm(s))  # Вывод: [0, 0, 1, 0, 3, 0, 1]
```

---

## Как это работает

- z[0] всегда 0 (по определению, обычно не используется).
- Если i попадает в текущее окно [l, r], используем уже посчитанные значения для ускорения.
- Если не попадает — находим совпадение “в лоб”.

---

## Применение

- Быстрый поиск всех вхождений шаблона P в тексте T: рассмотрим строку S = P + '$' + T, построим Z-массив для S, и если z[i] == len(P), то нашли вхождение.
- Поиск периодов строки, анализ повторов.

---

## Когда использовать

- Когда нужно быстро искать шаблоны в тексте
- Для анализа периодичности и повторов подстрок

---

## Когда не стоит использовать

- Для поиска очень длинных или сложных шаблонов (тогда эффективнее Aho-Corasick, префикс-функция Кнута-Морриса-Пратта и др.)

---

## Сложность

- **Время:** O(n)
- **Память:** O(n)
