# Дерево отрезков (Segment Tree)

---

## Описание задачи

**Дерево отрезков** (Segment Tree) — это структура данных, позволяющая за O(log n) быстро находить агрегатные значения (например, сумму, минимум, максимум) на произвольном отрезке массива и изменять элементы/диапазоны.

Используется для эффективной обработки запросов и изменений на отрезках (range queries и range updates).

---

## Подход (алгоритм)

1. Строим дерево, каждый узел которого содержит агрегатную информацию о некотором отрезке массива.
2. Быстро отвечаем на запросы к диапазону (например, сумма на отрезке [l, r]) и обновляем отдельные элементы или отрезки.

---

## Пример на Python (суммы на отрезке, point update)

```python
class SegmentTree:
    def __init__(self, data):
        self.n = len(data)
        self.size = 1
        while self.size < self.n:
            self.size *= 2
        self.tree = [0] * (2 * self.size)
        # Заполняем листья
        for i in range(self.n):
            self.tree[self.size + i] = data[i]
        # Строим дерево
        for i in range(self.size - 1, 0, -1):
            self.tree[i] = self.tree[2 * i] + self.tree[2 * i + 1]

    # Сумма на отрезке [l, r)
    def query(self, l, r):
        res = 0
        l += self.size
        r += self.size
        while l < r:
            if l % 2:
                res += self.tree[l]
                l += 1
            if r % 2:
                r -= 1
                res += self.tree[r]
            l //= 2
            r //= 2
        return res

    # Изменение элемента (индекс, новое значение)
    def update(self, idx, value):
        idx += self.size
        self.tree[idx] = value
        while idx > 1:
            idx //= 2
            self.tree[idx] = self.tree[2 * idx] + self.tree[2 * idx + 1]

# Пример использования:
arr = [1, 3, 5, 7, 9, 11]
st = SegmentTree(arr)
print(st.query(1, 5))  # Сумма с 1 по 4: 3+5+7+9 = 24
st.update(3, 10)       # arr[3] = 10
print(st.query(1, 5))  # 3+5+10+9 = 27
```

---

## Как это работает

- Листья дерева — элементы массива.
- Каждый внутренний узел — агрегат (например, сумма) двух дочерних интервалов.
- Запрос и обновление идут путём подъёма/спуска по дереву, затрагивая только нужные узлы.

---

## Применение

- Обработка запросов суммы/минимума/максимума на отрезке.
- Динамические задачи на массиве с изменениями.
- Задачи на статистику порядка, динамические RMQ, диапазонные обновления (с ленивым обновлением).

---

## Когда использовать

- Когда нужны быстрые запросы и модификации на отрезках.
- Когда Fenwick Tree недостаточен (например, нужны range update или сложная агрегатная функция).

---

## Когда не стоит использовать

- Для очень простых задач или маленьких массивов (проще перебор).
- Для задач только на префиксные суммы (лучше Fenwick Tree).

---

## Сложность

- **Время:** O(log n) на запрос и обновление
- **Память:** O(n)
