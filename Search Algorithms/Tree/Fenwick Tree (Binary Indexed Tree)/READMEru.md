# Дерево Фенвика (Fenwick Tree, Binary Indexed Tree)

---

## Описание задачи

**Дерево Фенвика** (или Binary Indexed Tree, BIT) — это структура данных, позволяющая эффективно (за O(log n)) выполнять две основные операции над массивом:
- Быстро находить сумму префикса (от начала до позиции i).
- Быстро изменять (обновлять) элемент массива.

Используется для задач на суммы отрезка, подсчёт инверсий, обработку запросов в онлайн-режиме и др.

---

## Подход (алгоритм)

Дерево Фенвика хранит массив частичных сумм, позволяя за O(log n) времени:
- Получить сумму на префиксе [1, i].
- Добавить значение к элементу (или изменить его).

В основе лежит работа с младшими значащими битами индексов (LSB, least significant bit).

---

## Пример на Python

```python
class FenwickTree:
    def __init__(self, size):
        self.n = size
        self.tree = [0] * (self.n + 1)  # индексация с 1

    def update(self, i, delta):
        while i <= self.n:
            self.tree[i] += delta
            i += i & -i  # переход к родителю

    def query(self, i):
        res = 0
        while i > 0:
            res += self.tree[i]
            i -= i & -i  # переход к следующему предку
        return res

    def range_sum(self, l, r):
        return self.query(r) - self.query(l - 1)

# Пример использования:
ft = FenwickTree(8)
for idx, val in enumerate([3,2,1,6,5,4,7,8], start=1):
    ft.update(idx, val)

print(ft.range_sum(3, 6))  # Сумма с 3 по 6: 1+6+5+4 = 16
```

---

## Как это работает

- Каждый элемент дерева отвечает за сумму некоторого диапазона элементов оригинального массива.
- Манипуляции с битами индекса позволяют быстро переходить к диапазонам (узлам дерева) при обновлении и запросе суммы.

---

## Применение

- Задачи на динамические суммы отрезков (без модификаций диапазона).
- Подсчёт инверсий.
- Онлайн-обработка запросов (например, сумма чисел на префиксе, добавление значения).
- Некоторые задачи на статистику порядка (k-й по счёту элемент, меньшие числа справа и т.п.).

---

## Когда использовать

- Когда нужны быстрые запросы суммы на префиксе и обновления отдельных элементов.
- Когда не требуется изменение значений на целых диапазонах (для этого лучше использовать дерево отрезков/Segment Tree).

---

## Когда не стоит использовать

- Если нужны запросы изменения/запроса суммы на произвольном отрезке (не только префикс) с поддержкой изменений целых отрезков — лучше использовать Segment Tree.
- Для очень маленьких массивов можно обойтись простым перебором.

---

## Сложность

- **Время:** O(log n) на обновление и запрос
- **Память:** O(n)
