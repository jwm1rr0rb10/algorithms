# Обзор алгоритмов поиска и структур данных на деревьях

Алгоритмы и структуры данных на деревьях — основа эффективного хранения, поиска и обработки иерархической информации. Они используются в индексировании, поиске, сжатии данных, диапазонных запросах, проверке орфографии и многом другом.

В этом руководстве сравниваются ключевые алгоритмы на деревьях, их сложности, практические применения, достоинства и подводные камни.

## Описание алгоритмов и применения

### **Поиск в бинарном дереве поиска (BST)**

- **Описание:** Хранит элементы в отсортированном порядке для быстрого поиска, вставки и удаления.
- **Когда применять:** Для динамичных, упорядоченных данных с быстрыми изменениями и запросами.
- **Плюсы:** O(log n) при балансе.  
- **Минусы:** O(n) для несбалансированных деревьев. Используйте AVL/красно-черные для баланса.

### **Сериализация/десериализация дерева**

- **Описание:** Преобразует дерево в строку/массив и обратно.
- **Когда применять:** Для сохранения/передачи структур дерева, задач на LeetCode и др.

### **Дерево Фенвика (BIT)**

- **Описание:** Эффективно поддерживает префиксные запросы и точечные обновления.
- **Когда применять:** Для кумулятивных частот, префиксных сумм.
- **Плюсы:** Проще и экономнее по памяти, чем дерево отрезков для сумм.
- **Минусы:** Менее гибкое, чем дерево отрезков.

### **Дерево отрезков**

- **Описание:** Гибкая структура для сложных диапазонных запросов и обновлений.
- **Когда применять:** Когда нужны диапазонные min/max/sum/и др. запросы и обновления.
- **Плюсы:** Поддерживает множество типов запросов и обновлений.
- **Минусы:** Сложнее в реализации, чем BIT.

### **Trie (префиксное дерево)**

- **Описание:** Хранит строки по символам в узлах.
- **Когда применять:** Для автодополнения, поиска слов, проверки орфографии, префиксных запросов.
- **Плюсы:** Быстрая вставка, поиск, префиксные запросы.
- **Минусы:** Высокое потребление памяти для больших/разрежённых алфавитов.

### **Lowest Common Ancestor (LCA)**

- **Описание:** Находит ближайшего общего предка двух вершин дерева.
- **Когда применять:** Запросы на иерархических данных, файловых системах, родословных.
- **Подходы:** Наивный O(n), бинарное поднятие O(log n), Эйлеров обход + RMQ O(1) после препроц.

### **Обходы дерева (in-order, pre-order, post-order)**

- **Описание:** Обход всех узлов дерева в определённом порядке.
    - **In-order:** Левый, корень, правый (отсортированный вывод для BST).
    - **Pre-order:** Корень, левый, правый (удобно для сериализации, копирования).
    - **Post-order:** Левый, правый, корень (удаление поддеревьев, вычисления).
- **Когда применять:** Для любой обработки, сериализации, копирования, печати, вычислений на дереве.

## Особенности и подводные камни

- **Балансировка:** Сбалансированные деревья обеспечивают предсказуемую производительность.
- **Память:** Trie и дерево отрезков могут занимать много памяти при больших/разрежённых данных.
- **Глубокий стек:** Рекурсия на несбалансированных деревьях может привести к переполнению стека.
- **Fenwick vs Segment Tree:** Fenwick проще и экономичнее для сумм, Segment Tree — для сложных диапазонных операций.

---

## Практические советы

- Для простых префиксных запросов используйте Fenwick tree.
- Для сложных диапазонных запросов и обновлений — Segment Tree.
- Trie для быстрого поиска по префиксу/слову (учитывайте память!).
- Сериализуйте и десериализуйте одним и тем же методом (BFS/DFS).
- Для быстрых LCA-запросов используйте бинарное поднятие или Эйлеров обход.
