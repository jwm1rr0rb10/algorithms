# Нахождение наименьшего общего предка (Lowest Common Ancestor, LCA)

---

## Описание задачи

**Наименьший общий предок** (Lowest Common Ancestor, LCA) двух узлов дерева — это такой узел, который является их общим предком и находится на наибольшей глубине (то есть "самый близкий" к обоим узлам).

Задача: по данному корню дерева и двум заданным узлам найти их LCA.

---

## Подходы

### 1. Рекурсивный способ для бинарного дерева

Работает для любого бинарного дерева (не обязательно бинарного поиска):

- Если текущий узел равен одному из искомых — возвращаем его.
- Рекурсивно ищем LCA в левом и правом поддеревьях.
- Если оба поддерева вернули не None, то текущий узел — LCA.

### Пример на Python

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

def lowest_common_ancestor(root, p, q):
    if not root or root == p or root == q:
        return root
    left = lowest_common_ancestor(root.left, p, q)
    right = lowest_common_ancestor(root.right, p, q)
    if left and right:
        return root
    return left if left else right

# Пример использования:
# Пусть p и q — это ссылки на узлы в дереве (не значения!)
```

---

## Расширенные методы

- Для большого количества LCA-запросов используют методы с препроцессингом: бинарное поднятие (binary lifting), RMQ по времени входа/выхода (Euler tour).
- В дереве поиска (BST) можно ускорить поиск, используя свойства BST.

---

## Применение

- Задачи на иерархию, родословные, файловые системы, анализ графов, задачи на дерево отрезков.
- В алгоритмах на графах и деревьях для поиска путей и расстояний.

---

## Когда использовать

- Когда нужно быстро и эффективно находить LCA для нескольких пар узлов.

---

## Когда не стоит использовать

- Для графов с циклами (это работает только для деревьев).
- Если нужно быстро отвечать на множество запросов — лучше использовать препроцессинг (binary lifting, RMQ).

---

## Сложность

- **Время:** O(n)Lowest Common Ancestor для одного запроса в произвольном дереве, O(log n) при препроцессинге.
- **Память:** O(n)
