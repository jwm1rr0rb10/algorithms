# Поиск в повернутом отсортированном массиве

## Что такое повернутый отсортированный массив?

**Повернутый отсортированный массив** — это отсортированный массив, который был циклически сдвинут (повёрнут) на некоторое число позиций.

**Пример:**  
Исходный: `[1, 2, 3, 4, 5, 6, 7]`  
После поворота: `[5, 6, 7, 1, 2, 3, 4]`

---

## Задача

Дан повернутый отсортированный массив и некоторое число. Нужно найти индекс этого числа или вернуть -1, если его нет.

---

## Почему обычный бинарный поиск не подходит?

Обычный бинарный поиск работает только для полностью отсортированного массива. После поворота порядок нарушается, но **одна из половин всегда остаётся отсортированной**.

---

## Решение: модифицированный бинарный поиск

На каждом шаге определяем, какая из половин отсортирована, и ищем внутри неё:

1. Задаём `left` и `right` — границы массива.
2. Пока `left <= right`:
    - Находим середину: `mid = (left + right) // 2`.
    - Если `arr[mid] == target` — возвращаем `mid`.
    - Проверяем, какая половина отсортирована:
        - Если `arr[left] <= arr[mid]` (левая половина отсортирована):
            - Если `target` в диапазоне `[arr[left], arr[mid]]`, ищем слева (`right = mid - 1`).
            - Иначе — справа (`left = mid + 1`).
        - Иначе (правая половина отсортирована):
            - Если `target` в `[arr[mid], arr[right]]`, ищем справа (`left = mid + 1`).
            - Иначе — слева (`right = mid - 1`).
3. Если не нашли — возвращаем -1.

---

## Сложность

- **Время:** O(log n)
- **Память:** O(1)

---

## Пример на Python

```python
def search_rotated_sorted_array(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        # Левая половина отсортирована
        if arr[left] <= arr[mid]:
            if arr[left] <= target < arr[mid]:
                right = mid - 1
            else:
                left = mid + 1
        # Правая половина отсортирована
        else:
            if arr[mid] < target <= arr[right]:
                left = mid + 1
            else:
                right = mid - 1
    return -1

# Пример использования
arr = [4, 5, 6, 7, 0, 1, 2]
print(search_rotated_sorted_array(arr, 0))  # Выведет: 4
print(search_rotated_sorted_array(arr, 3))  # Выведет: -1
```

---

## Итог

В повернутом отсортированном массиве можно эффективно искать элемент, модифицируя бинарный поиск: на каждом шаге определять, какая половина отсортирована, и фокусироваться на ней. Благодаря этому сохраняется сложность O(log n).
