# Поиск в двумерной матрице

## Условие задачи

Дана двумерная матрица с такими свойствами:
- Каждая строка отсортирована по возрастанию слева направо.
- Первый элемент каждой строки **строго больше** последнего элемента предыдущей строки.

**Пример:**
```
[
  [1, 3, 5, 7],
  [10, 11, 16, 20],
  [23, 30, 34, 50]
]
```

Дано число. Определите, содержится ли оно в матрице.

---

## Подход

Если “вытянуть” все элементы матрицы в один массив, он будет отсортирован.  
Значит, можно использовать бинарный поиск, не преобразуя матрицу явно.

### Алгоритм

1. Пусть `m` — количество строк, `n` — столбцов.
2. Задаём `left = 0`, `right = m * n - 1`.
3. Пока `left <= right`:
    - Считаем `mid = (left + right) // 2`.
    - Переводим индекс в координаты: `row = mid // n`, `col = mid % n`.
    - Сравниваем `matrix[row][col]` с целевым числом:
        - Если равно — возвращаем `True` (или индекс).
        - Если меньше — ищем правее.
        - Если больше — ищем левее.
4. Если не найдено — возвращаем `False`.

---

## Сложность

- **Время:** O(log(m * n))
- **Память:** O(1)

---

## Пример на Python

```python
def search_matrix(matrix, target):
    if not matrix or not matrix[0]:
        return False
    m, n = len(matrix), len(matrix[0])
    left, right = 0, m * n - 1
    while left <= right:
        mid = (left + right) // 2
        row, col = mid // n, mid % n
        num = matrix[row][col]
        if num == target:
            return True
        elif num < target:
            left = mid + 1
        else:
            right = mid - 1
    return False

# Пример использования
matrix = [
  [1, 3, 5, 7],
  [10, 11, 16, 20],
  [23, 30, 34, 50]
]
print(search_matrix(matrix, 3))   # Выведет: True
print(search_matrix(matrix, 13))  # Выведет: False
```

---

## Итог

Если представить двумерную матрицу как один отсортированный массив, можно использовать бинарный поиск и эффективно искать элементы за O(log(m * n)).
