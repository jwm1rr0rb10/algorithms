# Поиск прыжками (Jump Search)

## Что это такое

Jump Search — алгоритм поиска в **отсортированном массиве или списке**. Это компромисс между линейным поиском (простой, но медленный) и бинарным поиском (быстрый, делит пополам). Jump Search делит массив на блоки фиксированного размера и "перепрыгивает" через них, чтобы быстро определить нужный диапазон, а затем выполняет линейный поиск внутри найденного блока.

---

## Как работает

1. Пусть длина массива — `n`. Выбираем шаг (размер блока) `m` (обычно `m = √n`).
2. Начинаем с первого элемента и прыгаем вперёд на `m` элементов, проверяя последний элемент каждого блока.
3. Останавливаемся, когда элемент на конце блока **больше или равен** искомому, или доходим до конца массива.
4. Выполняем линейный поиск внутри найденного блока.
5. Если элемент найден — возвращаем его индекс, иначе возвращаем -1.

---

## Сложность

- **Время:** O(√n)
    - Максимум O(√n) прыжков и столько же сравнений внутри блока
- **Память:** O(1)
    - Используются только переменные для индексов

---

## Когда использовать

- Массив **отсортирован**
- Нужно что-то проще, чем бинарный поиск, но быстрее линейного
- Бинарный поиск по каким-то причинам невозможен (например, дорогой случайный доступ)

## Когда не использовать

- Массив **не отсортирован** (Jump Search не работает)
- Массив очень маленький (лучше линейный или бинарный)
- **Бинарный поиск** почти всегда лучше (O(log n) против O(√n)), используйте его, если можно

---

## Пример на Python

```python
import math

def jump_search(arr, target):
    n = len(arr)
    step = int(math.sqrt(n))
    prev = 0

    # Прыжки по блокам
    while prev < n and arr[min(step, n) - 1] < target:
        prev = step
        step += int(math.sqrt(n))
        if prev >= n:
            return -1

    # Линейный поиск внутри блока
    for i in range(prev, min(step, n)):
        if arr[i] == target:
            return i
    return -1

# Пример использования
arr = [1, 3, 5, 7, 9, 13, 17, 21, 29, 35, 42, 56]
print(jump_search(arr, 21))  # Выведет: 7
print(jump_search(arr, 6))   # Выведет: -1
```

---

## Итог

Jump Search — редко используемый алгоритм для отсортированных массивов. Он быстрее линейного поиска, но уступает по скорости бинарному. На практике почти всегда предпочтителен бинарный поиск.
