# ♾️ Divide and Conquer — Алгоритмы

Этот раздел содержит классические задачи и алгоритмы, решаемые методом **разделяй и властвуй**. Такой подход заключается в рекурсивном разбиении задачи на подзадачи меньшего размера, решении их отдельно и последующем объединении результатов.

---

## Где и когда применять

### 1. **Closest Pair of Points**
- **Где применять:** Геометрия, задачи про ближайшую пару точек на плоскости.
- **Лучше D&C:** Почти единственный эффективный способ для O(n log n).
- **Где хуже:** Для очень малого n проще перебор.

### 2. **Convex Hull (D&C version)**
- **Где применять:** Обработка изображений, GIS, задачи поиска выпуклой оболочки.
- **Лучше D&C:** Для больших наборов точек, когда требуется высокая эффективность.
- **Где хуже:** При малом числе точек возможно проще алгоритмы Грэхема или Джарвиса.

### 3. **Maximum Subarray (D&C version)**
- **Где применять:** Когда хочется показать D&C-подход или модифицировать задачу.
- **Лучше D&C:** В теоретических целях, для параллельных решений.
- **Где хуже:** На практике Кадане (O(n)) проще и быстрее для одномерного массива.

### 4. **Count Inversions**
- **Где применять:** Оценка "беспорядка" в массиве, теория сортировок, биоинформатика.
- **Лучше D&C:** Когда нужно эффективно считать инверсии за O(n log n).
- **Где хуже:** Для маленьких массивов лучше перебор.

### 5. **Kth Element (QuickSelect)**
- **Где применять:** Быстрый поиск k-го по порядку элемента (например, медианы).
- **Лучше D&C:** Для больших массивов, без необходимости полной сортировки.
- **Где хуже:** В худшем случае — медленно, хотя на практике редко случается.

### 6. **Majority Element (D&C version)**
- **Где применять:** Когда важна демонстрация D&C или нужны обобщения.
- **Лучше D&C:** В учебных целях.
- **Где хуже:** На практике Boyer-Moore проще и быстрее (O(n), O(1) памяти).

### 7. **Karatsuba Algorithm**
- **Где применять:** Умножение очень больших чисел (криптография, научные вычисления).
- **Лучше D&C:** Для длинной арифметики, где простое умножение слишком долгое.
- **Где хуже:** Для маленьких чисел обычное умножение быстрее.

### 8. **Strassen’s Matrix Multiplication**
- **Где применять:** Быстрое перемножение больших квадратных матриц.
- **Лучше D&C:** Для действительно больших матриц (на практике — тысячи на тысячи).
- **Где хуже:** Для маленьких — стандартное умножение проще и быстрее; также требует квадратных матриц размера 2^n.

## Итоги

- **Divide and Conquer** — мощный универсальный подход для сложных задач, дающий выигрыш по скорости и расширяемости.
- На практике для небольших размеров проще и быстрее классические/жадные/динамические методы.
- Важно понимать ограничения по памяти и особенности реализации для конкретной задачи.
