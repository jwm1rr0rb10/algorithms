# Динамическое программирование на графах: объяснение и пример

## Что такое ДП на графах?

Динамическое программирование на графах — это подход для решения задач оптимизации и подсчёта на графах (особенно на ациклических, DAG), разбивая их на более простые подзадачи.  
Часто используется для поиска кратчайших/длиннейших путей, подсчёта количества путей, а также для других рекурсивных задач, где состояние зависит от предыдущих результатов.

---

## Как работает ДП на графах?

- **Состояние:** Обычно `dp[v]` хранит ответ для вершины `v` (например, длина максимального пути, заканчивающегося в `v`).
- **Переход:** Для каждой вершины `v` обновляем значение на основе её предшественников (или соседей), например:  
  `dp[v] = max(dp[u] + weight(u, v))` для всех `u`, таких что есть ребро из `u` в `v`.
- **Порядок:** Вершины обрабатываются в топологическом порядке, чтобы все зависимости были рассчитаны до вычисления состояния.

---

## Пример на Python (длиннейший путь в DAG)

```python
from collections import defaultdict, deque

def longest_path_dag(n, edges):
    # Построение графа
    graph = defaultdict(list)
    indegree = [0] * n
    for u, v, w in edges:
        graph[u].append((v, w))
        indegree[v] += 1

    # Топологическая сортировка
    queue = deque([i for i in range(n) if indegree[i] == 0])
    topo_order = []
    while queue:
        node = queue.popleft()
        topo_order.append(node)
        for neighbor, _ in graph[node]:
            indegree[neighbor] -= 1
            if indegree[neighbor] == 0:
                queue.append(neighbor)

    # Инициализация DP
    dp = [float('-inf')] * n
    dp[0] = 0  # Пусть 0 — стартовая вершина

    for u in topo_order:
        for v, w in graph[u]:
            if dp[u] + w > dp[v]:
                dp[v] = dp[u] + w

    return dp

# Пример использования
n = 5
edges = [
    (0, 1, 3),
    (0, 2, 2),
    (1, 2, 1),
    (1, 3, 4),
    (2, 3, 2),
    (3, 4, 1)
]
print(longest_path_dag(n, edges))  # Ответ: [0, 3, 3, 7, 8]
```

---

## Сложность

- **Время:** O(V + E), где V — количество вершин, E — количество рёбер
- **Память:** O(V + E)

---

## Где используется?

- Планирование задач с зависимостями (проект-менеджмент, сборочные системы)
- Подсчёт или оптимизация путей в DAG
- Решение учебных/курсовых задач с условиями-предпосылками
- Некоторые задачи в AI и играх

---

## Когда использовать ДП на графах?

- Задача сводится к подзадачам на графе
- Граф — ациклический (DAG) или циклы можно учесть/обработать
- Необходим эффективный подсчёт или оптимизация по путям/состояниям

Программное обеспечение для управления проектами (например, диаграммы Ганта или build-системы) использует ДП на DAG для расчёта минимального времени завершения комплекса зависимых задач.
