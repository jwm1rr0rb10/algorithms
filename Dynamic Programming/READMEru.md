# Обзор алгоритмов динамического программирования

Динамическое программирование (ДП) — мощная техника для решения задач, которые разбиваются на пересекающиеся подзадачи с сохранением их решений. ДП применяется в задачах оптимизации, последовательностей и комбинаторики.

В этом разделе приведены популярные алгоритмы динамического программирования, их временные и пространственные сложности, а также рекомендации по выбору подхода для разных сценариев.

---

## Таблица сложности алгоритмов

| Алгоритм                        | Временная сложность   | Пространственная сложность |
|----------------------------------|----------------------|---------------------------|
| Coin Change (монеты)             | O(n * amount)        | O(amount)                 |
| DP на графах                     | O(V + E)             | O(V + E)                  |
| DP на деревьях                   | O(n)                 | O(n)                      |
| Edit Distance (редакционное расстояние) | O(m * n)     | O(m * n)                  |
| House Robber (ограбление домов)  | O(n)                 | O(1)                      |
| Knapsack (рюкзак)                | O(n * W)             | O(n * W)                  |
| Longest Common Subsequence       | O(m * n)             | O(m * n)                  |
| Longest Increasing Subsequence   | O(n * log n)         | O(n)                      |
| Longest Palindromic Substring    | O(n²)                | O(n²)                     |
| Matrix Chain Multiplication      | O(n³)                | O(n²)                     |
| Memoization (мемоизация)         | O(n)                 | O(n)                      |
| Paint House                      | O(n)                 | O(n)                      |
| Palindrome Subarray              | O(n²)                | O(n²)                     |
| Palindrome Subsequence           | O(n²)                | O(n²)                     |
| Partition problem (разбиение)    | O(n * sum)           | O(n * sum)                |
| Tabulation (табуляция)           | O(n * sum/2)         | O(n * sum/2)              |
| Word Break                       | O(n³)                | O(n)                      |

---

## Когда применять эти алгоритмы

- **Оптимизационные задачи:** Knapsack, Coin Change, Matrix Chain Multiplication — классика для поиска максимума/минимума при ограничениях.
- **Выравнивание последовательностей:** Edit Distance, Longest Common Subsequence — биоинформатика, сравнение текстов, проверка орфографии.
- **Подпоследовательности и подстроки:** Longest Increasing Subsequence, Longest Palindromic Substring/Subsequence — анализ данных и паттернов.
- **Разбиения и декомпозиции:** Partition Problem, Word Break — справедливое деление, аллокация ресурсов, обработка текста.
- **Деревья и графы:** DP на деревьях и графах — для иерархических и сетевых структур.
- **Memoization vs Tabulation:** Мемоизация — для рекурсивных задач с выборочным вычислением подзадач. Табуляция — для итеративных, где можно явно построить таблицу.

---

## Где эти алгоритмы особенно хороши

- **Эффективность:** Позволяют снизить экспоненциальную сложность до полиномиальной.
- **Универсальность:** Типовые паттерны DP подходят к множеству задач.
- **Ясность:** Код часто отражает логику рекурсии задачи.

## Когда стоит быть осторожным

- **Память:** Некоторые ДП-алгоритмы (например, рюкзак, умножение матриц) требуют много памяти. Используйте оптимизацию пространства, если возможно.
- **Огромные пространства состояний:** Если состояний слишком много, DP может быть неэффективен (например, Knapsack с большим W).
- **Не всегда самый быстрый:** Для некоторых задач (например, Longest Increasing Subsequence) жадные или бинарные методы быстрее классического ДП.

---

## Практические советы

- **Дизайн состояния:** Минимизируйте размерность и избыточность DP-состояний.
- **Оптимизация памяти:** Используйте одномерные массивы и сжатие состояний, где это возможно.
- **Tabulation vs Memoization:** Табуляция (bottom-up) обычно быстрее из-за отсутствия рекурсии и проще для оптимизации по памяти.
- **Отладка:** Выводите промежуточные DP-таблицы для проверки корректности.
