# Палиндромная подпоследовательность — Динамическое программирование

## Описание задачи

Дана строка (или массив), требуется найти самую длинную палиндромную подпоследовательность.  
**Подпоследовательность** может пропускать символы, но сохраняет исходный порядок.

Варианты:
- Найти длину самой длинной палиндромной подпоследовательности.
- Восстановить саму подпоследовательность.

---

## Идея алгоритма и подход

- Используем динамическое программирование:
  - Пусть `dp[i][j]` — длина самой длинной палиндромной подпоследовательности в `s[i..j]`.
  - Если `s[i] == s[j]`, то `dp[i][j] = dp[i+1][j-1] + 2`.
  - Иначе: `dp[i][j] = max(dp[i+1][j], dp[i][j-1])`.
  - Для подстрок длины 1: `dp[i][i] = 1`.
- Заполняем DP-таблицу для всех подстрок.

---

## Пример на Python: длина самой длинной палиндромной подпоследовательности

```python
def longest_palindrome_subseq(s):
    n = len(s)
    dp = [[0]*n for _ in range(n)]
    for i in range(n-1, -1, -1):
        dp[i][i] = 1
        for j in range(i+1, n):
            if s[i] == s[j]:
                dp[i][j] = dp[i+1][j-1] + 2
            else:
                dp[i][j] = max(dp[i+1][j], dp[i][j-1])
    return dp[0][n-1]

# Пример использования:
s = "bbbab"
print(longest_palindrome_subseq(s))  # Вывод: 4 ("bbbb")
```
